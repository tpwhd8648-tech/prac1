# 레이디저스티스 스펙 수정 및 두께 미확인 행 숨김 인수인계 문서

작성일: 2026-06-22

## 1. 목표

레이디저스티스의 발행처/발행국 불일치(시트 vs 코드)를 공급처 데이터 기준으로
정정하고, 두께가 미확인인 상품에서 "확인 중" 문구가 화면에 노출되지 않도록
상품정보 탭 표시 방식을 개선한다.

## 2. 현재 상태

**완료 (push 반영 확인)**

- `coin-descriptions.js` — 레이디저스티스 발행처/스펙 정정 (커밋 `6782155`)
  - mint: 게르마니아 민트(Germania Mint) → **스코츠데일 민트 (Scottsdale Mint)**
  - country: 독일 → **니우에 (Niue)**
  - diameter: 32.69mm → **31.75mm**
  - thickness: 2.87mm → **2.4mm**
  - detail 설명 문구도 "독일 게르마니아 민트가 발행하는" → "니우에에서 발행되고
    스코츠데일 민트가 제조하는"으로 함께 수정
  - 사용자 제공 공급처 데이터(Metal/Weight/purity/Issuing Country/Manufacturer/
    Sizes 등) 기준으로 확인 완료

- `coin-descriptions.js` + `coin-detail.html` — 두께 미확인 항목 표시 방식 변경
  (커밋 `f4df54a`)
  - 기존: `specs.thickness`가 없으면 `'확인 중'` 텍스트를 강제로 표시
  - 변경: `coin-detail.html`의 상품정보 탭 렌더링을
    `${specs.thickness ? '<tr>...두께...</tr>' : ''}` 형태로 수정해,
    값이 없으면 두께 행 자체를 표에서 생략
  - `coin-descriptions.js`에서 체코라이언·케이브라이언 두 항목의
    `thickness: '확인 중'` 필드를 완전히 삭제 (데이터 자체에 더미 문구를
    남기지 않음)
  - 다른 17개 코인은 thickness 값이 있어 정상적으로 두께 행이 노출됨

## 5. 참고 사항

- **두께 미확인 항목 처리 패턴**: `coin-descriptions.js`의 `specs` 객체에
  `thickness` 필드 자체를 넣지 않으면, `coin-detail.html`이 자동으로 그 행을
  표에서 생략한다. 더미 값("확인 중" 등)을 데이터에 넣을 필요가 없음 —
  앞으로 다른 스펙 필드(직경, 연도 등)도 같은 패턴을 쓰려면 `coin-detail.html`의
  해당 `||` fallback 로직도 같이 조건부 렌더링으로 바꿔야 함 (현재 직경/연도는
  여전히 `|| '확인 중'` fallback이 남아있어 패턴이 통일돼 있지 않음)
- **레이디저스티스 스펙 오류 원인 추정**: 기존 코드의 mint/country/diameter/
  thickness 값이 성조지 등 영국 Royal Mint 시리즈와 동일했던 것으로 보아,
  코인 추가 시 다른 항목을 복붙한 뒤 일부 필드만 수정하고 나머지를 놓친
  것으로 추정됨. 신규 상품 추가 시 복붙 후 전체 필드 재검토 필요
- **체코라이언 발행국 주의**: 제조사는 Czech Mint(체코)지만 발행국은
  **니우에**임 — 계속 혼동 주의 (스펙표에 발행처/발행국 불일치 의심 표시로
  사용자에게도 안내함)
- **앞면/뒷면 기준**: 주제 그림(동물·건물 등) = 앞면(`.png`), 인물 초상·국가
  문장 = 뒷면(`-back.png`)
- **스펙 필드 구조**: `year`, `mint`, `country`, `purity`, `weight`,
  `diameter`, `thickness`(선택), `condition`, `verified`
- **신규 상품 추가/삭제 시**: `coin-descriptions.js` + `products.js`의
  IMAGE_MAP + 시트 행 3곳을 함께 관리
