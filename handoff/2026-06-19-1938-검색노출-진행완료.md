## 5. 참고 사항

- **레포 이름이 곧 도메인**: GitHub Pages는 레포 이름이 `사용자명.github.io`일 때 호스트 루트(`/`)에서 바로 서비스된다. 지금처럼 일반 레포(`prac1`)를 쓰던 상태에서 이름만 바꿔주면 별도 레포 생성 없이 루트 호스팅이 가능 — 새 레포 만드는 것보다 간단한 해결책이었음.
- **네이버 소유확인 URL 입력 시 주의**: "사이트 등록"은 호스트 단위(`https://tpwhd8648-tech.github.io`, 경로 없이)로 입력해야 하지만, **"사이트맵 제출" 입력칸은 반대로 전체 URL**(`https://tpwhd8648-tech.github.io/sitemap.xml`)을 넣어야 한다. 두 입력 방식이 다르니 헷갈리지 않도록 주의 (이번 세션에서 실제로 한번 잘못 안내했던 부분).
- **robots.txt 보류 이유**: 기본값(파일 없음)이 "전체 허용"이라 robots.txt를 굳이 만들어도 차이가 없음. 다만 `v2.html`/`v3.html` 같은 테스트 페이지가 우연히 검색에 노출되는 걸 막고 싶어지면 그때 `Disallow`로 추가하면 됨.
- **push 워크플로우**: 토큰은 매 세션 새로 받아야 하며(저장 안 함), 권한 확인 → 변경사항 요약 → 사용자 명시적 확인("푸쉬해줘") → push → `git ls-remote`로 반영 확인 순서로 진행 중. 이번 세션 커밋: `b2f1011`(네이버 확인 파일), `196e954`(sitemap.xml).
- v2.html / v3.html은 연습용 파일 (이전 문서에서 이월된 메모, 계속 유효함)
