## 5. 참고 사항

- **images 폴더는 sparse-checkout에서 제외되어 있음**: `git sparse-checkout set --no-cone '/*' '!/images'`로 클론하기 때문에 `images/`는 기본적으로 작업 디렉토리에 안 보인다. 이미지 작업이 필요하면 (1) `codeload.github.com`으로 전체 zip을 받아 압축 작업하고, (2) 실제 레포에 반영할 때는 `git add --sparse images/`로 스테이징해야 한다 (`git add images/`만 쓰면 "sparse-checkout 밖" 오류 발생). GitHub API(`api.github.com/repos/.../git/trees`)는 비로그인 상태에서 rate limit(시간당 60회)에 자주 걸리므로 이미지처럼 큰 디렉토리는 zip 다운로드가 더 안정적.
- **이미지 압축 방식 선택 이유**: 처음엔 PNG→JPEG 변환(용량을 더 줄일 수 있음)을 검토했으나, `script.js`의 상품-파일명 매핑 배열과 `coin-detail.html`의 `buildImageList()` 함수에 `.png` 확장자가 하드코딩돼 있어서 확장자를 바꾸면 코드 여러 곳을 같이 고쳐야 하고 실수 위험이 컸다. 그래서 확장자·해상도는 그대로 두고 압축률만 높이는 pngquant 방식(무손실에 가까움)을 택함 — 코드 변경 없이 결과만 적용 가능했음.
- **web_fetch 도구 제약(재확인)**: 이번 세션에서도 v2.html/v3.html처럼 한 번도 검색/접속된 적 없는 URL은 직접 fetch가 안 됨을 재확인. 이런 건 사용자가 브라우저로 직접 확인하는 게 가장 빠름.
- **GitHub 토큰 사용 기록**: 이번 세션에 사용자가 토큰을 줘서 이미지 압축 커밋(`7e7f33a`)을 push함. 지침대로 토큰 값은 응답에 노출하지 않았고 메모리/파일에도 저장하지 않음.
