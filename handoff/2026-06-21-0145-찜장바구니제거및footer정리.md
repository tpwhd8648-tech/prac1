**완료 (push 후 `git ls-remote`로 원격 반영 검증함)**

- 찜한 상품 / 장바구니 사이드바 제거 (커밋 `b36f5c1`)
  - `mypage.html`: "찜한 상품(WISHLIST)" 섹션 제거. 사이트 어디에도 찜하기

    (주석으로 사유 기록).
  - `coins.html`, `index.html`: 장바구니 사이드바 HTML, toast div 제거.
    `addToCart()`를 호출하는 "담기" 버튼이 사이트 어디에도 없었고,
    사이드바 안의 "구매 문의하기" 버튼도 onclick 핸들러가 없어 완전히
    죽은 UI였음.
  - `nav.js`: 헤더의 장바구니 아이콘 버튼 제거.
  - `script.js`: 미사용 cart/toast 함수(`openCart`/`closeCart`/`addToCart`/
    `removeFromCart`/`updateCartUI`/`getCardPrice`/`showToast`) 제거.
  - `style.css`: `.cart-count`, `.cart-overlay`, `.cart-sidebar`,
    `.cart-header` 등 CART 블록 전체, `.btn-checkout`, `.toast` 관련 CSS
    제거.
  - 사용자 결정으로 구매 문의는 `coin-detail.html`의 기존 단일
    "구매 문의하기"(카카오톡 오픈채팅 연결) 버튼으로 통일 — 별도 신규
    구현 없이 기존 버튼을 그대로 유지하는 것으로 충분하다고 판단.
  - `node --check`로 script.js/nav.js 문법 정상 확인, 전체 레포에서
    cart/toast 관련 죽은 참조(`openCart`, `addToCart`, `cart-sidebar`,
    `cart-count`, `showToast` 등)가 더 남아있지 않은지 grep으로 전수 확인.
  - `.btn-cart`/`.btn-cart-wrap` 클래스는 이름은 비슷하지만 별개로,
    상품 카드의 "상품 보기" 버튼 스타일로 여전히 사용 중이라 그대로 유지함
    (삭제하지 않음).

- `footer.js`의 죽은 `id="footer-locations"` 제거 (커밋 `72792fc`)
  - 가리키는 링크가 사이트 어디에도 없던 죽은 id만 제거. `class=
    "footer-locations"`는 `style.css`에서 실제 스타일링에 쓰이고 있어
    그대로 유지.
  - `node --check footer.js` 문법 정상 확인.

  없었음). 추가 조치 불필요.

## 5. 참고 사항

- 장바구니/찜 기능이 다시 필요해지면, 커밋 `b36f5c1` 직전 버전의
  `mypage.html`/`coins.html`/`index.html`/`nav.js`/`script.js`/
  `style.css`를 참고해 복원 가능.
- 사이트의 실제 구매 흐름은 "장바구니 결제"가 아니라 상품 상세페이지
  (`coin-detail.html`)에서 카카오톡 오픈채팅으로 연결되는 "건별 구매
  문의" 방식임. 향후 새 기능을 설계할 때 이 모델을 기준으로 판단할 것.
- `index.html`에 이미 작성돼 있는 `.badge-stock`/`.badge-sameday`/
  `.btn-sameday` 등 CSS는 실제 적용 안 된 채 남아있음 (뱃지 기능
  자체를 안 하기로 했으므로 죽은 CSS이지만, 이번 세션 범위 밖이라 그대로
  둠 — 정리하려면 별도 작업으로 진행).
- `footer-locations`는 이제 id 없이 class만 남음. 향후 페이지 내 앵커가
  필요해지면 새 id를 부여해서 사용할 것 (이 div를 재사용해도 무방).
- GitHub PAT 토큰: 이번 세션은 매번 새로 입력받아 push해서 토큰 재사용
  이슈 없었음. 세션 시작 시 직전 세션 토큰 revoke 여부 확인하는 습관은
  계속 유지할 것.
