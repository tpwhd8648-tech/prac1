**(1) 인덱스 캐러셀 모바일 세로 스크롤 수정 (커밋 `b7fe7b6`)**
- 증상: 손가락이 카드 위에 있을 때 세로로 스와이프하면 카드가 위쪽으로 살짝 밀리는 현상
- 어제 접근(커밋 `841871c`): `.product-card`에 `touch-action: pan-y` 추가 → 실패
  - 진단이 틀렸음. "touch-action 비상속" 문제로 접근했으나 실제 원인이 아니었음
- 오늘 접근: `#products-scroll-wrap`에 `overflow-y: hidden` 추가 → 성공 (기기 확인 완료)
  - 실제 원인: `overflow-x: auto`만 있고 `overflow-y`가 기본값 `visible`이라 세로 방향으로 카드가 밀릴 공간이 존재했던 것
  - `overflow-y: hidden` 한 줄로 세로 방향을 물리적으로 차단 → JS/touch-action 무관하게 해결

## 5. 참고 사항

- **캐러셀 overflow-y 교훈**: 가로 스크롤 컨테이너에 `overflow-x: auto`만 걸면 `overflow-y`가 `visible` 상태가 되어 세로 방향으로 콘텐츠가 밀릴 수 있음. 향후 가로 스크롤 컨테이너 추가 시 `overflow-y: hidden`도 함께 명시할 것.
- **어제 touch-action 작업(커밋 `841871c`)**: `.product-card`에 `touch-action: pan-y`가 추가된 상태로 남아 있음. 오늘 `overflow-y: hidden`으로 근본 해결됐으므로 해당 코드는 무해하지만 엄밀히는 불필요한 상태. 제거 여부는 선택사항.
- **솔드아웃 관련 코드 분산 위치** (수정 시 4곳 항상 함께 점검):
  1. `scripts/generate-coin-pages.js` (SSG 템플릿)
  2. `script.js` (index.html 런타임 가격 갱신)
  3. `coins.html` 인라인 스크립트 (런타임 가격 갱신 + 검색 재렌더링)
  4. `products.js` (coins.html 검색 시 카드 렌더 함수)
