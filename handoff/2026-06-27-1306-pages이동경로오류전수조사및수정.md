# pages/ 이동 경로 오류 전수조사 및 수정 인수인계 문서

작성일: 2026-06-27 (KST)

## 1. 목표

직전 세션(다른 계정, 핸드오프 미작성)에서 루트 HTML 6개를 `pages/` 폴더로
이동하는 대규모 구조 변경이 있었음. 이번 세션은 그 변경 이후 실제로 깨진
부분이 있는지 코드 기준으로 전수조사하고, 발견된 버그를 모두 수정·배포·재검증함.

## 2. 현재 상태

  대한 핸드오프 문서가 작성되지 않은 상태**였음 (확인 완료: `git log --oneline`
  으로 커밋 메시지/시각 대조).
- 그래서 문서를 신뢰하지 않고 레포 실물 코드를 기준으로 전수조사 진행.

### 발견 및 수정한 버그 3건 (모두 확인 완료, 커밋 `f435f2c`)

**(1) [영향 높음] `pages/coins.html` 상품 카드 클릭 시 전부 404**
- 원인: 인라인 스크립트의 `COINS_CARD_OPTIONS.linkTemplate`이
  `coins/coin-${slug}.html` / `coin-detail.html?name=...`로 `../` 없이
  생성됨. `pages/coins.html`은 depth 1이라 실제로는
  `/pages/coins/coin-xxx.html`(존재하지 않음)로 풀림.
- SSG가 만든 초기 정적 카드는 `../coins/...`로 정상이었으나, 페이지 로드 후
  구글 시트 재조회로 그리드를 통째로 덮어쓰는 `renderProducts()`가 실행되면
  전부 깨진 링크로 교체됨 — 실사용자에게는 항상 404였음.
- 수정: `pages/coins.html` 599번 줄에 `../` 추가.

**(2) [영향 보통] 코인 상세페이지 20개 — BreadcrumbList가 옛 URL(`/coins.html`) 참조**
- 원인: `scripts/generate-coin-pages.js`의 BreadcrumbList JSON-LD 템플릿이
  `${SITE_URL}/coins.html`(이동 전 경로)을 그대로 사용 — `coins/coin-*.html`
  20개 전부 동일하게 오염.
- 화면 링크는 정상이라 사용자 클릭에는 영향 없으나, 구글 검색 리치 결과의
  빵 부스러기 구조화 데이터가 404 URL을 가리키는 상태였음.
- 수정: `${SITE_URL}/pages/coins.html`로 템플릿 수정 + 기존 생성된 20개
  파일도 sed로 동기화.

**(3) [영향 낮음] `pages/*.html` 6개 — favicon 링크에 `../` 누락**
- `coins.html, contact.html, gold-price.html, mypage.html, privacy.html,
  terms.html` 모두 `href="favicon.svg"`로 되어 있어 실제로는
  `/pages/favicon.svg`(없음)를 가리킴.
- `nav.js`가 런타임에 자체 favicon(`data:` URI)을 head에 추가 주입하므로
  화면상 파비콘은 정상 표시될 가능성이 높아 사용자 체감 영향은 적었음.
  다만 불필요한 404 요청 + 코드 일관성 문제로 함께 정리.
- 수정: 6개 파일 모두 `href="../favicon.svg"`로 변경.

   출력과 일치함을 교차 검증)

## 5. 참고 사항

- **`pages/` 폴더(depth 1) 관련 상대경로 체크리스트**: 앞으로 `pages/*.html`을
  수정할 때는 (1) `<link>`/`<script src>`의 `../assets/...` (2) 코인 상세 이동
  링크의 `../coins/...`, `../coin-detail.html` (3) favicon `../favicon.svg`
  세 가지를 항상 같이 점검할 것. 이번 버그 3건 모두 "SSG/공용 JS는 고쳤는데
  페이지 안에 손으로 박아둔 별도 코드는 못 고친" 패턴이었음.
- **nav.js의 favicon 런타임 주입**: `nav.js`가 로드 시 `data:` URI 기반
  favicon `<link>`를 head에 추가로 꽂기 때문에, `<head>`에 정적으로 박힌
  favicon href가 깨져 있어도 화면상으로는 잘 안 드러남. 이런 "JS가 가려주는"
  버그는 grep으로 경로만 봐서는 못 잡고 실제 동작(스크립트 로드 순서·런타임
  덮어쓰기)까지 추적해야 발견됨 — 비슷한 사례 또 있을 수 있으니 의심되면
  JS 주입 로직까지 같이 확인할 것.
- **SSG 재실행으로 수정 검증하는 법**: 본 작업 환경(sandbox)은
  `docs.google.com`이 네트워크 허용 목록에 없어 `generate-coin-pages.js`를
  로컬에서 직접 실행해 테스트할 수 없음. 대신 push 후 GitHub Actions
  `workflow_dispatch`로 실제 실행시키고, 그 결과(diff 없음 = 수정본과 일치)로
  검증하는 방식을 사용함. 앞으로도 SSG 관련 수정은 이 방식으로 검증 가능.
