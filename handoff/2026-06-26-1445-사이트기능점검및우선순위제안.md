# 사이트 기능 점검 및 우선순위 제안 인수인계 문서

작성일: 2026-06-26

## 1. 목표

전수조사(코드 오류/충돌 없음 확인) 이후, 사이트에 추가하면 좋을 기능과 법적으로
필수인 항목을 코드 기준으로 점검하고 우선순위를 정리.

## 2. 현재 상태

### 점검 방법
모든 항목은 코드를 직접 grep/view로 확인한 결과이며, 추측이 아님.

### 🔴 거의 필수 (법적 리스크) — 확인 완료

**(1) 사업자 정보 — "숨김"이 아니라 가짜 정보가 실제로 노출 중**
- `footer.js` 66번 줄: 가짜 사업자등록번호(`123-45-67890`)가 **화면에 그대로 노출**됨
  (주석상으로는 "확정 전까지 숨김"이라고 되어 있으나 실제로는 숨겨진 게 아님)
- `contact.html` 309번 줄: `"서울특별시 종로구 종로 1가 1번지 (임시 주소)"`가 **"(임시 주소)"라는
  표시까지 달려서 화면에 노출 중**
- 전자상거래법상 통신판매업 신고 시 사업자등록번호·통신판매업신고번호·대표자명·주소·
  고객센터 연락처 표시가 의무사항. 카카오톡 상담 기반이어도 웹에서 가격·상품을 노출하고
  거래를 유도하면 통신판매업으로 간주될 가능성 높음
- **이건 코드 문제가 아니라 사업 운영 단계의 문제** — 실제 사업자 등록 진행 상태를
  먼저 확인 필요. 등록 전이라면 가짜 정보를 노출하는 현재 상태가 등록 후보다
  오히려 더 위험할 수 있음 (허위 정보 노출)

**(2) 이용약관 / 개인정보처리방침 페이지 — 확인 완료, 없음**
- `ls *.html`로 확인 — terms/privacy/policy/약관/개인정보 관련 페이지 자체가 존재하지 않음
- `footer.js` 34번 줄 주석: "서비스 약관 / 자주 묻는 질문: 연결할 실제 페이지가 아직 없어"
  → 이미 인지하고 있던 보류 항목
- `auth.js`에서 `createUserWithEmailAndPassword`로 이메일+비밀번호를 실제로 수집 중
  (Firebase Auth 회원가입 기능 작동 중) → 개인정보처리방침이 법적으로 필요한 시점

### 🟡 있으면 전환율에 직접 도움 — 확인 완료

**(3) 가격 정렬 기능 없음**
- `coins.html`에 카테고리/브랜드/재고 필터는 있으나 가격 낮은순/높은순 정렬 코드 없음 (grep 결과 0건)

**(4) 최근 본 상품 / 추천 상품 확장**
- "당일수령 가능 금화" 추천(`renderRelatedCoins`)은 있으나, 개인화된 "최근 본 상품"은 없음
- 구현 비용 대비 효과는 중간

**(5) 위시리스트(찜하기)**
- `mypage.html` 134~136번 줄에 주석으로 명시: "찜하기 버튼이 사이트 어디에도 없어
  진입점이 없는 기능이라 섹션 자체를 제거함. 추후 도입 시 재추가 가능" — 이미 인지된 보류 항목

**(6) 가격 알림 (목표가 도달 시 알림)**
- 관련 기능 코드 전혀 없음 (grep 결과 0건)
- 이메일/푸시 발송 인프라가 별도로 필요해 구현 비용 큼

### 🔵 검토해볼 만함 (우선순위 낮음) — 확인 완료

**(7) 리뷰/평점**
- 리뷰 UI, `AggregateRating`/`Review` JSON-LD 스키마 모두 없음 (grep 결과 0건)
- 카카오톡 상담 기반 거래라 후기 수집 구조 자체가 약함

**(8) FAQ 별도 페이지 분리**
- 현재 JSON-LD는 `Product`, `BreadcrumbList`만 있고 `FAQPage` 스키마 없음
- FAQ가 각 코인 상세페이지 "구매안내" 탭(4개 아코디언) 안에 코인마다 반복 존재
- 별도 페이지로 분리 시 `FAQPage` 구조화 데이터 적용 가능 → SEO 효과

**(9) 다국어(영문) 버전**
- 현재 `<html lang="ko">`, 도메인/제목/설명 전부 한국어 시장 타겟
- 해외 수요 확인 전까지 우선순위 낮음

### 추가로 점검된 항목 (전수조사 연장선)
- TradingView 차트, 검색 자동완성, 관련 코인 추천 등은 이미 잘 갖춰져 있음
- 검색 결과 0건 시 안내 없이 조용히 전체 상품으로 전환되는 UX 빈틈 발견
  (`coins.html`의 `matched = false` 분기 — 안내 배너 없음)
- 단위 환산 계산기(1돈=3.75g 등) 없음
