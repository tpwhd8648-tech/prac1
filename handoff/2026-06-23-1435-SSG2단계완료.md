## 1. 목표

index.html과 coins.html의 상품 카드를 GitHub Actions 빌드 시 정적으로
미리 주입해, 크롤러(Naver Yeti 등)가 JS 실행 없이도 상품 목록을 볼 수
있도록 한다. (SSG 2단계)

## 2. 현재 상태

**완료 (push 반영 및 직접 확인)**

- **`generate-coin-pages.js` 확장** (커밋 `883ad47`):
  - `renderProductCard()`, `renderIndexGrid()`, `renderCoinsGrid()`,
    `injectGrid()` 함수 추가
  - `fetchSheetRows()`에 `visible`(G열), `order`(H열) 컬럼 추가
  - `main()`에 index.html / coins.html 그리드 주입 로직 추가

- **index.html / coins.html SSG 마커 삽입** (동 커밋):
  - `<!-- SSG_GRID_START -->` / `<!-- SSG_GRID_END -->` 마커 추가

- **버그 수정 2건**:
  1. **`visible` 빈 문자열 버그** (커밋 `dc8827e`):
     시트 G열이 빈 칸일 때 `visible = ""`로 파싱돼 필터에서 전부
     걸러지던 문제 수정. `String(v ?? '').toLowerCase() || 'all'` 방식으로
     빈 문자열도 `'all'`로 폴백하도록 변경.
  2. **워크플로우 `git add` 누락** (커밋 `60ff63b`):
     `.github/workflows/generate-coin-pages.yml`의 커밋 단계에서
     `coin-*.html sitemap.xml`만 add하고 `index.html`과 `coins.html`이
     빠져 있어 카드가 커밋되지 않던 문제 수정.

- **Actions 실행 및 실제 주입 확인** (커밋 `508c259`):
  - index.html: 카드 8개 주입 (`visible = 'all'` 상품)
  - coins.html: 카드 20개 주입 (`visible = 'all'` 또는 `'coins'` 상품)
  - 클릭 링크: `coin-{slug}.html` 개별 페이지로 직접 연결
  - `visible = 'none'`인 행(코뿔소, 세인트조지드래곤)은 정상적으로 제외됨

**시트 visible 값 정리**:
- `all` → index.html + coins.html 양쪽에 노출
- `coins` → coins.html에만 노출
- `main` → index.html에만 노출 (현재 시트에 해당 행 없음)
- `none` → 어디에도 미노출 (코뿔소, 세인트조지드래곤)
- 빈 칸 → `'all'`로 처리 (버그 수정 후)

## 5. 참고 사항

- **SSG 2단계 동작 방식**: Actions 실행(매일 KST 00:00 또는 수동) 시
  시트 데이터를 읽어 카드 HTML을 생성하고 마커 사이에 주입. 브라우저에서
  JS가 로드되면 `updateCardPricesFromSheet()`가 `.card-price` 요소만
  실시간으로 갱신. 크롤러는 정적 HTML 그대로 카드를 읽을 수 있음.
- **visible 컬럼 기본값**: G열이 없거나 빈 칸이면 `'all'`로 처리됨.
  시트에 새 행 추가 시 `visible` 값을 명시하지 않아도 양쪽 페이지에 노출됨.
- **Actions 로그 직접 조회 불가**: 워크플로우 상세 로그는 Azure Blob Storage
  도메인에 저장되어 이 작업 환경에서 접근 불가. 대신 `raw.githubusercontent.com`
  에서 실제 배포된 파일을 직접 가져와 검증하는 방식 사용.
