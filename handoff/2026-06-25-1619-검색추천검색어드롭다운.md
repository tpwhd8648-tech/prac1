# 검색 추천검색어 드롭다운 추가 인수인계 문서

작성일: 2026-06-25

## 1. 목표

검색창에 코인 이름 일부를 입력하면 추천 코인명이 드롭다운으로 표시되도록 구현한다.
기존 `상품 검색...` placeholder도 더 명확한 문구로 교체한다.
데스크탑 전용 기능으로, 모바일은 현행 유지(검색창 자체가 hidden).

## 2. 현재 상태

### 2-1. 구현 완료

- **placeholder 변경**: `"상품 검색..."` → `"금화 이름으로 검색 (예: 메이플, 버팔로)"`
- **추천검색어 드롭다운**: 입력 시 `COIN_NAMES` 배열과 부분 일치 항목을 드롭다운으로 표시
- **키보드 지원**: ↑↓ 방향키로 항목 이동, Enter로 선택, Esc로 닫기
- **닫기 처리**: 항목 클릭, 외부 클릭, blur 시 자동 닫힘 (`blur` → `setTimeout(150ms)`, `mousedown`에 `preventDefault`로 순서 충돌 방지)

### 2-2. 변경 파일

- `nav.js` (CSS 추가, HTML 수정, JS 이벤트 로직 교체)

### 2-3. 검증

- `node --check nav.js` 통과
- `suggestBox` null 안전 처리 확인
- `position: relative` — `style.css`의 `.header-search`에 이미 선언되어 있어 드롭다운 절대위치 정상
- 기존 `custom-dropdown` (카테고리 선택)과 클래스명/ID 완전 분리 확인
- 모바일 영향 없음 (`@media max-width:768px`에서 `.header-search { display:none }`)

### 2-4. 미확인 (실사이트 확인 필요)

- 드롭다운 렌더링 위치가 시각적으로 자연스러운지 (flex 컨테이너 안 absolute 포지션)
- 실제 코인명 입력 시 추천 항목 정상 표시 여부

## 5. 참고 사항

- **`COIN_NAMES` 위치**: `nav.js` 검색창 이벤트 블록 내 정적 배열. 상품 추가/삭제 시 수동으로 동기화 필요.
- **드롭다운 클래스명**: `.search-suggest` / `.search-suggest-item` — 기존 `.custom-dropdown` / `.custom-dropdown-item`과 별개. 혼용 금지.
- **blur/mousedown 순서**: 항목 클릭 시 `blur`가 먼저 발생해 드롭다운이 닫힐 수 있으므로 `mousedown`에 `e.preventDefault()` 처리함. 수정 시 이 순서 유지 필요.
- **모바일 검색 미구현**: 모바일 검색창 추가는 별도 작업으로 보류. 필요 시 다음 세션에서 진행.
