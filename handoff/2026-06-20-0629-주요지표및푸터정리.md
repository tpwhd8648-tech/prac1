## 2. 현재 상태

**완료(확인 완료 — push 후 `git ls-remote`로 원격 반영 검증함)**

- `gold-price.html`
  - 기존 금/은/백금 차트 아래에 "주요지표 더보기" 토글 버튼 추가. 클릭 시
    원유(WTI)/비트코인(BTC)/환율(USD/KRW) 탭이 펼쳐지고, 탭 클릭 시 큰 차트
    하나로 전환되는 방식(기존 금/은/백금 탭과 동일한 UX).
    - 심볼: WTI=`TVC:USOIL`, BTC=`BITSTAMP:BTCUSD`, FX=`FX_IDC:USDKRW`
  - 차트는 패널을 처음 펼칠 때만 로드(`extraChartLoaded` 플래그로 중복 로드 방지).
  - 메인 탭(금/은/백금)과 더보기 탭(원유/비트코인/환율)은 `closest('.chart-tabs')`
    로 그룹이 분리되어 있어 서로 영향 없음(직접 클릭 테스트로 확인).
  - 안내 문구("※ 시세는 TradingView...")를 기존 위치(메인 차트 바로 아래)에서
    "주요지표 더보기" 섹션 전체 아래(페이지 맨 끝)로 이동. 토글 패널 바깥에 있어
    더보기를 펼치지 않아도 항상 노출됨.
  - id 중복 없음 확인(`grep -o 'id="[^"]*"' | sort | uniq -c`로 검증).
- `footer.js`
  - "언어" 섹션(한국어/English/繁體中文 셀렉터, 클릭해도 동작 안 하던 죽은 UI)
    전체 삭제. 다른 푸터 항목(강남 지점 정보, 카탈로그 죽은 링크, 소셜 아이콘 등)
    은 이번 세션에서 손대지 않음.
  - `node --check footer.js`로 문법 정상 확인.

**의사결정(코드 변경 없음, 기록용)**

- 다국어(영어/중국어) 풀 페이지 번역 작업은 진행하지 않기로 결정. 목적이
  "가끔 오는 외국인 손님이 대충 읽을 수 있으면 됨" 수준이라, Chrome/Safari
  내장 자동번역으로 충분하다고 판단함. URL 폴더 방식(`/en/`, `/zh/`)이나 JS
  토글 방식 둘 다 보류 — 향후 "해외 검색 유입(SEO)"이 목적으로 바뀌면 그때
  URL 폴더 방식부터 재검토할 것(자동번역은 SEO 효과가 없으므로).
- 가상자산 추가 지표는 비트코인만 유지, 이더리움 추가는 사용자가 일단 보류함.
  필요시 `BITSTAMP:ETHUSD` 심볼로 더보기 탭에 추가 가능.
- 달러인덱스(DXY, `TVC:DXY`), 미국 10년물 금리(`TVC:US10Y`) 등 금값 연동
  지표 추가도 제안만 하고 보류 상태.

**남아있는 사소한 항목**

- `style.css`의 `.lang-section`, `.lang-options`, `.lang-opt` 규칙은 더 이상
  쓰이는 곳이 없는 죽은 CSS. 화면엔 영향 없지만, footer.js 다른 정리 작업할
  때 같이 지우면 됨.

**보안 메모**

- 이번 세션에서 push에 사용한 GitHub PAT가 사용자에 의해 대화 중 총 3회
  평문으로 재입력됨(직전 세션 문서 1회 포함하면 그 이상). 매번 revoke를
  권고했으나 미확인. 다음 세션 시작 시 토큰이 아직 살아있는지, revoke했는지
  먼저 확인할 것.

## 5. 참고 사항

- gold-price.html의 메인 탭 그룹과 더보기 탭 그룹은 코드상 `switchChart` /
  `switchExtraChart` 두 함수로 분리되어 있고, 각각 `btn.closest('.chart-tabs')`
  로 자기 그룹만 active 토글함. 새 탭을 추가할 때 이 패턴을 따라야 다른 그룹에
  영향이 안 감.
- 더보기 패널의 차트는 지연 로딩(lazy load)이라 페이지 첫 진입 시 TradingView
  위젯이 1개만 생성됨(메인 차트). 더보기를 펼쳐야 2번째 위젯이 생성되므로
  성능에 유리함 — 새 지표 추가 시에도 이 lazy-load 패턴 유지 권장.
- GitHub PAT 토큰 관련: 사용자가 같은 토큰을 여러 세션/턴에 걸쳐 반복
  재사용 중. push 자체는 매번 성공했지만, 보안상 매 세션 시작 시 한 번씩
  상기시켜줄 것 (대화 기록에 토큰이 평문으로 여러 번 남아있는 상태).
- 다국어 관련해서 향후 사용자가 "해외 검색 유입을 늘리고 싶다"고 마음이
  바뀌면, JS 토글 방식이 아니라 URL 폴더 방식(`/en/`, `/zh/`)부터 검토할 것
  — 이번 세션에서 SEO 효과 차이를 이미 설명했으므로 같은 논의를 반복하지
  않아도 됨.
