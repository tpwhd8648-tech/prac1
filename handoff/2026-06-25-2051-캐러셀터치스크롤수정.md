# 캐러셀 터치 스크롤 수정 인수인계 문서

## 1. 목표

인덱스 메인페이지 상품 캐러셀에서 모바일로 손가락을 대고 세로로 움직일 때
페이지가 스크롤되지 않고 캐러셀이 가로로 스크롤되던 문제를 수정한다.

## 2. 현재 상태

### 완료된 작업 (이번 세션)

**(1) 전수조사 (버그 없음 확인)**
- 스크립트 로드 순서, IMAGE_MAP ↔ coin-*.html, canonical/og:url, 깨진 앵커,
  SHEET_ID 일치, sitemap 커버리지 등 전항목 이상 없음
- `coin-detail.html`의 canonical → `coins.html`은 의도된 설계 (SSG fallback 전용)
- `index.html` smoothScroll 중복 선언, `coins.html` applyTabFilter 선언 순서
  → 둘 다 실제 동작 문제 없음 (스코프 격리 + 호이스팅으로 안전)

**(2) 캐러셀 touch-action 수정 (커밋 `841871c`)**
- 원인: `#products-scroll-wrap`(overflow-x:auto)에 `touch-action:pan-y`가 있었으나
  실제 터치가 닿는 자식 요소 `.product-card`에는 `touch-action`이 없었음
  → `touch-action`은 CSS 비상속 속성이라 부모 설정이 자식에 전달 안 됨
  → 브라우저가 카드(touch-action:auto)에서 모든 방향 터치를 처리하려 해서
    세로 드래그가 페이지 스크롤 대신 캐러셀 가로 스크롤로 해석됨
- 수정: `index.html` 인라인 `<style>`의 `#products-grid .product-card`에
  `touch-action: pan-y` 추가 → index.html 전용 (coins.html 등 미적용)
- `coins.html` 등 다른 페이지는 가로 스크롤 컨테이너 자체가 없어 원래부터 문제 없음

**(3) 잘못된 수정 → 즉시 복구**
- 세션 중 `pan-y` → `pan-x`로 잘못 바꿨다가 즉시 `pan-y`로 복구 완료 (커밋 `689dd49`)
- 현재 `#products-scroll-wrap`과 `.brand-grid` 모두 `touch-action:pan-y` 상태 (정상)

## 5. 참고 사항

- **touch-action 비상속 주의**: `touch-action`은 CSS 상속이 안 됨. 컨테이너에만 주면
  안에 있는 자식 요소에는 적용 안 됨. 터치가 실제로 닿는 가장 안쪽 요소에 직접 줘야 함.
- **인덱스 전용 스타일**: 이번 수정은 `index.html` 인라인 `<style>`의
  `#products-grid .product-card`에만 적용. `style.css` 건드리지 않음.
  다른 가로 스크롤 컨테이너 추가 시 동일 패턴 적용할 것.
- **솔드아웃 관련 코드 분산 위치** (수정 시 4곳 항상 함께 점검):
  1. `scripts/generate-coin-pages.js` (SSG 템플릿)
  2. `script.js` (index.html 런타임 가격 갱신)
  3. `coins.html` 인라인 스크립트 (런타임 가격 갱신 + 검색 재렌더링)
  4. `products.js` (coins.html 검색 시 카드 렌더 함수)
