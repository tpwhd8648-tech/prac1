## 5. 참고 사항

- **JSON-LD price를 "100% 실시간"이 아니라 "빌드 시점 시세"로 박은 이유**:
  화면 표시 가격은 여전히 nav.js가 브라우저에서 시트를 실시간 fetch해
  갱신하지만(변경 없음), 검색엔진(구글 리치 스니펫)이 보는 JSON-LD에
  `price` 필드가 없으면 구글이 Offer 자체를 무효 처리하는 것으로 알려져
  있어, "어제~오늘 시세 기준이라도 박는" 쪽을 택함. 매일 1회(KST 00:00)
  Actions 빌드가 도므로 최대 1일 지연이 생기고, `priceValidUntil`로
  시세가 변동될 수 있음을 명시함.
- **가격 계산 공식 일치 원칙**: `generate-coin-pages.js`의 price 계산은
  반드시 `nav.js`/`products.js`의 화면 표시 공식
  (`Math.round(krwPerOz × premium / 1000) × 1000`)과 동일하게 유지해야
  함 — 둘이 어긋나면 검색결과에 노출된 가격과 실제 화면 가격이 달라지는
  문제가 생김. 향후 화면 가격 계산 로직(nav.js)을 고치면
  `generate-coin-pages.js`의 `renderCoinPage()`도 같이 고쳐야 함.
- **krwPerOz 출처**: 시트 `계산` 탭의 row[0](금시세 USD/oz) ×
  row[4](환율 KRW/USD). `nav.js`의 `updateNavPrices()`와 동일한 컬럼
  인덱스를 그대로 가져다 씀 — 시트 컬럼 순서가 바뀌면 두 곳 다 같이
  고쳐야 함.
- **GitHub Actions 로그 직접 다운로드 불가**: 워크플로우 상세 로그는
  Azure Blob Storage(`productionresultssa0.blob.core.windows.net`)에
  저장되는데, 이 작업 환경 네트워크 허용 목록에 없어 직접 조회 불가
  (`x-deny-reason: host_not_allowed`). 대신 GitHub API로 워크플로우/
  잡/스텝 상태(success/failure)는 확인 가능하고, 실제 결과물은
  `raw.githubusercontent.com`(허용 도메인)에서 생성된 파일을 직접
  가져와 검증하는 방식으로 우회함. 사용자에게 직접 Actions 탭 확인을
  요청할 필요가 줄어듦.
- **GitHub API 비인증 요청은 레이트리밋 걸림**: 토큰 없이 `api.github.com`
  호출 시 IP 기준 레이트리밋에 빠르게 도달함 — Actions 실행 결과 확인이
  필요하면 토큰을 받아서 인증 요청으로 처리할 것.
- **PAT**: 이번 세션에서 push 권한 정상 확인(`admin/maintain/push: true`).
  토큰은 세션 작업에만 사용, 메모리/파일/로그에 저장하지 않았고 응답에도
  노출하지 않음 (마스킹 처리).
- **SSG 전체 구조**: 이제 코인 정적 페이지 생성(Actions) + coins.html
  링크 연결 + 화면 가격 연동 + JSON-LD price/availability까지 전부
  완료. 신규 코인 추가 시: `coin-descriptions.js` + `products.js`의
  IMAGE_MAP + 시트 행 3곳 추가하면 다음 Actions 실행 시 자동으로
  `coin-{slug}.html`(가격/재고 포함)까지 생성됨.
- **앞면/뒷면 기준**: 주제 그림(동물·건물 등) = 앞면(`.png`), 인물
  초상·국가 문장 = 뒷면(`-back.png`)
- **스펙 필드 구조**: `year`, `mint`, `country`, `purity`, `weight`,
  `diameter`, `thickness`(선택), `condition`, `verified`
