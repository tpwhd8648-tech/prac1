## 5. 참고 사항

- 검색 기능: `nav.js`의 `goSearch(keyword)` 함수가 데스크탑/모바일 검색
  양쪽에서 공용으로 쓰임. 새로 검색 입력 지점을 추가할 때 이 함수를
  재사용하면 됨. 빈 입력값은 `coins.html`(전체 상품)로 보내도록 처리되어
  있어 별도의 빈값 처리가 필요 없음.
- `coins.html`의 `?q=` 필터 로직(`matchesKeyword`)은 한글 합성어 오매칭을
  막기 위해 키워드 앞뒤 글자가 한글이면 별개 단어로 보고 제외하는 방식임 —
  검색 관련 로직을 수정할 때 이 부분을 깨뜨리지 않도록 주의.
- footer.js에서 삭제한 항목들은 전부 화면에는 안 보이지만 코드 내 HTML
  주석으로 "왜 지웠는지 + 실제 정보/페이지가 생기면 다시 추가하라"는 메모를
  남겨둠. 다음 세션에서 footer.js를 열어보면 어떤 항목이 "임시 삭제"
  상태인지 주석만 보고 바로 파악 가능함.
- "오시는 길" 링크는 별도 페이지가 아니라 같은 페이지 내 위치 정보 섹션으로
  앵커 이동하는 방식(`#footer-locations`)을 택함 — 별도 오시는 길 페이지를
  만들 필요가 생기면 이 앵커 링크를 실제 페이지 링크로 교체할 것.
- GitHub PAT 토큰: 사용자가 매 push마다 채팅에 토큰을 평문으로 입력 중.
  push 자체는 매번 성공했지만, 세션 시작 시 매번 revoke 여부를 확인해줄 것.
