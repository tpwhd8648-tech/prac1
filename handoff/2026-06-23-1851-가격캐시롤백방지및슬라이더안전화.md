### 🔴 가격 캐시 롤백 위험 해소
- `coin-detail.html`은 이미 `sessionStorage`의 `midas_price_cache`를
  직접 읽는 방식으로 개선되어 있었으나, `scripts/generate-coin-pages.js`
  (정적 `coin-*.html` 20개를 생성하는 스크립트)는 구버전 — `#tb-gold`/
  `#tb-rate` DOM `textContent`를 파싱하는 방식 그대로였음.
- `generate-coin-pages.js`의 485~491번 줄을 `coin-detail.html`과 동일한
  `sessionStorage.getItem('midas_price_cache')` 직접 읽기 방식으로 교체.
- 커밋 `1f421c0`으로 push → `scripts/generate-coin-pages.js`가 GitHub
  Actions의 push 트리거 대상 파일이라 **즉시 자동 실행됨** → 커밋
  `8fe6cb9`(github-actions[bot])로 `coin-*.html` 20개 전부 자동
  재생성/재커밋됨. `coin-eagle-2026.html` 등 샘플로 새 로직 반영 확인.

### 🟡 슬라이더 초기화 타이밍 안전화
- 기존: `script.js` 로드 즉시(전역 스코프) `querySelectorAll('.slide')`/
  `('.dot')`로 DOM을 캡처. `index.html`의 인라인 스크립트가 `.slide`를
  동적으로 먼저 채워 넣는 구조라, 현재는 로드 순서(인라인 스크립트가
  먼저, `script.js`는 `</body>` 직전)상 우연히 안전했지만 구조가
  바뀌면(`script.js`를 head로 옮기거나 defer 적용 등) 빈 NodeList를
  캡처해 슬라이더가 조용히 멈추는 위험이 있었음.
- `initHeroSlider()` 함수로 캡슐화해 DOM 조회를 "로드 시점"이 아닌
  "호출 시점"으로 늦추고, `document.readyState` 분기로
  `DOMContentLoaded` 이후 또는 즉시 실행되도록 처리. `.hero-slider`/
  슬라이드/버튼이 없으면 조용히 종료하는 가드도 추가.
- 다른 파일과의 변수/함수명 충돌(`track`, `currentSlide`, `goToSlide`
  등) 및 외부 참조 여부 grep으로 전부 확인 — 충돌 없음.
- 커밋 `369b651`로 push 완료.

### push 과정에서 발생한 정상 동작 (참고)
- 첫 push(`1f421c0`) 직후 Actions가 자동으로 `8fe6cb9` 커밋을 만들어
  원격에 먼저 push했고, 그 사이 로컬은 `1f421c0`에 머물러 있어 두 번째
  작업 커밋 시점에 "로컬이 origin보다 1커밋 뒤처짐" 상태가 발생함.
  fast-forward(`git merge origin/main --ff-only`)로 안전하게 동기화 후
  `369b651`을 그 위에 얹어 push. 커밋 그래프가 완전히 일직선이라 분기·
  머지 충돌 없음을 그래프와 raw 파일 diff로 확인 완료.

## 5. 참고 사항

- **이번 세션에서 종결된 항목**: 이전 핸드오프의 ⚙️ 기능/인프라 관련
  항목 중 "🔴 가격 캐시 롤백 위험"과 "🟡 슬라이더 타이밍"은 이번
  세션에서 완전히 해소됨 (코드 수정 + push + Actions 반영 + byte-diff
  검증까지 완료). 다음 세션에서 다시 꺼낼 필요 없음.
- **`generate-coin-pages.js` 수정 시 주의**: 이 파일은 GitHub Actions의
  push 트리거 대상(`coin-descriptions.js`, `products.js`,
  `scripts/generate-coin-pages.js` 중 하나)이므로, 이 파일을 push하면
  곧바로 Actions가 돌아 `coin-*.html` 20개 + `sitemap.xml` +
  `index.html`/`coins.html`이 자동 재생성/재커밋된다. push 직후 로컬이
  origin보다 뒤처지는 게 정상이니, 다음 작업 전에 반드시
  `git fetch && git merge origin/main --ff-only`로 동기화할 것.
- **`coin-detail.html`과 `generate-coin-pages.js`의 가격 로직은 이제
  완전히 동일한 패턴**(`sessionStorage`의 `midas_price_cache` 직접
  읽기)이므로, 앞으로 한쪽만 수정하고 다른 쪽을 놓치는 사고를 막기
  위해 가격 관련 로직을 고칠 때는 항상 양쪽 다 grep으로 대조할 것.
- **`script.js`의 `initHeroSlider()` 패턴**: `.hero-slider`가 없는
  페이지에서 `script.js`를 로드해도 안전하게 종료되므로, 이 함수
  도입으로 인한 부작용은 없음. 다만 `script.js`는 현재 `index.html`
  에만 로드되므로 실질적 영향 범위는 그 페이지로 한정됨.
- **GitHub PAT**: 이번 세션에서 두 차례 사용 (push 2회). 만료일
  미확인 — 다음 push 시 만료 여부 먼저 확인.
