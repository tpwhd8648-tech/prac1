# 이미지 폴더 구조 재편 인수인계 문서

작성일: 2026-06-26

## 1. 목표

이미지 파일 수 증가에 대비해 `images/` 루트에 혼재하던 상품 이미지를
`images/products/{year}/` 연도별 하위 폴더로 정리.
향후 2027, 2028년 상품 추가 시 파일 관리가 명확해지도록 구조 개선.

## 2. 현재 상태

### 완료된 작업 (이번 세션)

**(1) images/ 폴더 구조 재편 (커밋 `b0e7ca2`)**
- `images/products-{name}-{year}.png` (40개) → `images/products/{year}/{name}.png`
- 대상: 2025년 1종(레이디저스티스), 2026년 19종, 앞면/뒷면 각각
- `brands-*`, `header-*` 파일은 루트 유지 (연도 개념 없음)

**(2) coin-data.js IMAGE_MAP 경로 수정 (커밋 `b0e7ca2`)**
- `file: 'products-maple-2026.png'` → `file: 'products/2026/maple.png'`
- 20개 항목 전부 신규 형식으로 변경

**(3) generate-coin-pages.js slug 로직 2곳 수정 (커밋 `b0e7ca2`)**
- `buildCoinEntries()` (121번 줄): slug 정규식 교체
- `getSlugFromImageMap()` (832번 줄): 동일 정규식 교체
- 변경 전: `entry.file.replace(/^products-/, '').replace(/\.png$/, '')`
- 변경 후: `const m = entry.file.match(/products\/(\d{4})\/(.+)\.png$/i); return m ? \`${m[2]}-${m[1]}\` : ...`
- slug 결과값은 기존과 동일 (예: `maple-2026` → `coin-maple-2026.html`)

**(4) coins.html getSlugForProduct() 수정 (커밋 `b0e7ca2`)**
- 브라우저 런타임 카드 렌더링 시 slug 추출 함수 동일 정규식으로 교체

**(5) 5개 고정 페이지 og:image·twitter:image 경로 수정 (커밋 `b0e7ca2`)**
- index.html, coins.html, contact.html, gold-price.html, mypage.html
- `images/products-maple-2026.png` → `images/products/2026/maple.png`

**(6) coin-detail.html og:image 수정 (커밋 `7b05935`)**
- 1차 수정에서 누락됐던 coin-detail.html og:image·twitter:image 경로 수정

**(7) GitHub Actions 자동 재생성 완료**
- push 후 `Generate coin static pages` 워크플로우 자동 트리거
- coin-*.html 20개 + coins.html + index.html 정적 카드 전부 신규 경로로 재생성 확인

**(8) 구글 시트 image 컬럼 수정 (👤 직접 완료)**
- 20개 행의 image 컬럼을 신규 경로 형식으로 변경 완료

### 검증 결과 (확인 완료)
- coin-*.html 21개 이미지 참조 → 실제 파일 전부 존재 ✅
- IMAGE_MAP 20개 앞면/뒷면 파일 → 전부 존재 ✅
- slug ↔ coin-*.html 파일명 → 전부 일치 ✅
- 구 경로(`images/products-*`) 잔존 → 완전 제거 ✅

### 교훈
- 이번 작업에서 slug 추출 패턴이 3곳에 분산돼 있었음 (`buildCoinEntries`, `getSlugFromImageMap`, `coins.html getSlugForProduct`). 1차 검토 문서에서 2곳을 놓쳤고 다른 계정 검토로 발견됨.
- 앞으로 경로·패턴 변경 시 수정 전 `grep -rn` 전수 확인을 먼저 수행할 것.

## 5. 참고 사항

- **새 연도 상품 추가 시 이미지 규칙**: `images/products/2027/{name}.png` / `images/products/2027/{name}-back.png` 형태로 추가. `coin-data.js` IMAGE_MAP에 `file: 'products/2027/{name}.png'` 형식으로 추가하면 slug·경로 전부 자동 처리됨.
- **slug 추출 정규식**: `entry.file.match(/products\/(\d{4})\/(.+)\.png$/i)` → `${m[2]}-${m[1]}`. 이 패턴이 3곳에 동일하게 적용됨 (`buildCoinEntries`, `getSlugFromImageMap`, `coins.html getSlugForProduct`). 향후 경로 형식 변경 시 3곳 모두 함께 수정 필요.
- **coin-detail.html은 SSG fallback 전용**: 시트 image 컬럼값으로 `buildImageList()`가 경로를 구성함. 시트 image 컬럼 = `products/2026/maple.png` 형태 유지 필요.
- **솔드아웃 관련 코드 분산 위치** (수정 시 4곳 항상 함께 점검):
  1. `scripts/generate-coin-pages.js` (SSG 템플릿)

  3. `coins.html` 인라인 스크립트 (런타임 가격 갱신 + 검색 재렌더링)
  4. `products.js` (coins.html 검색 시 카드 렌더 함수)
