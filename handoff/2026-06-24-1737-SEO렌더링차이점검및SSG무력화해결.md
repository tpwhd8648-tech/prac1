# SEO 렌더링 차이 점검 및 SSG 무력화 해결 인수인계 문서

작성일: 2026-06-24

## 1. 목표

"손님이 어색하게 느낄 부분"을 전수 점검하다 발견한 판다 코인 발행처
표기 불일치를 수정하고, 이어서 "SEO 렌더링 차이"(크롤러가 보는
정적 HTML과 JS 실행 후 화면이 달라지는 문제)를 점검·해결한다.

## 2. 현재 상태

### 2-1. 판다 코인 발행처 표기 통일 (완료, 커밋 `6ac688a`)

- **발견**: `coins.html` 카드에는 "People's Bank of China"(중앙은행,
  발행 승인 기관)로, `coin-panda-2026.html` 상세페이지에는
  "China Gold Coin Incorporation"(실제 조폐 책임 기관)으로 서로
  다른 발행처명이 표시되고 있었음. 나머지 19개 코인은 전부 실제
  조폐 기관명(Royal Mint, Perth Mint, US Mint 등)으로 통일돼 있어
  판다만 예외였음.
- **조치**: 사용자 결정에 따라 "China Gold Coin"으로 통일.
  - `coins.html` 카드 텍스트 직접 수정 → push (`6ac688a`)
  - 구글 시트 brand 컬럼(판다 행)도 사용자가 직접 "China Gold Coin"으로 수정
  - GitHub Actions 트리거 → `coin-panda-2026.html` 재생성 확인
    (JSON-LD brand name, 상품정보 탭 product-brand-label 모두
    "China Gold Coin"으로 일치 확인 완료)

### 2-2. SEO 렌더링 차이 — index.html/coins.html의 SSG 무력화 구조 발견 및 해결 (완료, 커밋 `143af32`)

**발견된 문제**: GitHub Actions가 `index.html`/`coins.html`의
`<!-- SSG_GRID_START -->` 영역에 상품 카드(상품명/브랜드/이미지/링크)를
정적으로 미리 생성해두지만, 페이지 로드 시 `script.js`의
`loadProducts()`가 구글 시트를 다시 fetch해 `products-grid.innerHTML`을
**통째로 새로 그려서 SSG 카드를 덮어쓰는 구조**였음. 즉 SSG로 만든
정적 콘텐츠가 JS 실행 직후 사라지고 거의 동일한 내용으로 재생성되는
패턴이라, SSG의 SEO 의도가 사실상 무력화되고 있었음. (반면
`coin-detail.html` 기반 20개 코인 상세페이지는 이미 완전한 정적
페이지라 이 문제가 없음 — 영향 범위는 메인/목록 페이지 2개로 한정.)

**대안 비교 및 결정**:
| | A안(가격만 갈아끼움) | B안(데이터 달라질 때만 갈아끼움) |
|---|---|---|
| 구현 복잡도 | 낮음 | 높음 |
| SEO 안정성 | 항상 안전 | 평상시 안전, 시트 변경 시점엔 위험 남음 |
| 새 코인/순서 즉시 반영 | 불가(Actions 재빌드 필요) | 가능 |

→ 사용자가 시트 수정이 즉시 반영 안 되는 점을 "검수 단계로서 오히려
안전"하다고 판단, **A안으로 결정**.

**적용 내용**:
- `script.js`: `loadProducts()`/`renderProducts()`(및 의존 변수
  `currentKrwPrice`, `productsData`, `MAIN_CARD_OPTIONS`) 제거.
  카드는 SSG 정적 상태 그대로 유지하고, `updateCardPricesFromSheet()`
  (nav.js가 실시간 시세 수신 시 호출)가 가격 텍스트만 갈아끼움.
  스크롤 페이드인 애니메이션은 SSG 카드에 직접 재적용하도록 조정.
- `coins.html`: 검색 쿼리(`?q=...`)가 없으면 `loadProducts()`를
  호출하지 않음 — 평소 방문 시 SSG 카드가 그대로 유지됨. 검색
  쿼리가 있을 때만 시트를 fetch해 필터링된 결과로 다시 그림.
  카테고리 탭 필터는 기존에도 SSG 카드에 `display` 토글만 적용하는
  방식이었어서 영향 없음(이번 수정과 무관하게 이미 안전했음).

**검증**: JS 문법(`node --check`) 통과, HTML 구조(`<script>` 태그
짝, BeautifulSoup 파싱) 정상, diff로 의도한 2개 파일 외 변경 없음
확인. **단, 실제 브라우저 화면에서 가격 갱신·검색·탭 필터 동작은
검증 도구(화면 캡처) 부재로 직접 확인하지 못함 — 사용자가 실제
사이트에서 1회 확인 필요.**

**알려진 부수효과(의도된 트레이드오프)**:
- 시트에서 코인 추가/삭제, `visible`/`order` 값 변경 시 GitHub
  Actions 재빌드 전까지 화면에 즉시 반영되지 않음(이전에는 즉시
  반영됐음). 사용자가 이 트레이드오프를 인지하고 동의함.
- `index.html`의 `<script src="products.js">`가 이제 죽은
  의존성으로 남음(더 이상 `createProductCard`/`fetchProductsFromSheet`
  사용 안 함) — 동작에는 문제없으나 향후 dead code 정리 대상.

### 2-3. 구글/네이버 SEO 등록 현황 확인 (완료, 코드 점검만)

사용자가 "네이버만 등록한 것 같고 구글은 안 한 것 같다"고 문의 →
점검 결과 **구글도 이미 등록·인증 완료 상태**임을 확인:
- 구글: `index.html`/`coins.html` `<head>`에 `google-site-verification`
  메타태그 존재(메타태그 방식 인증)
- 네이버: `naverabf800e7bb5e01d2154c2f1ebf449eab.html` 파일 존재(파일
  업로드 방식 인증)
- 인증 방식이 달라(메타태그 vs 파일) 구글 쪽 흔적이 코드에서 덜
  눈에 띄었을 뿐, 둘 다 동등하게 등록·인증된 상태.
- robots.txt에 sitemap 경로 명시 확인(`Sitemap: https://tpwhd8648-tech.github.io/sitemap.xml`).
- 실제 색인/노출 여부는 직접 검증 불가(서치 콘솔 로그인 필요) —
  외부 웹 검색으로 사이트 노출 여부를 확인해봤으나 검색 결과에
  전혀 잡히지 않음(신규/소규모 사이트의 일반적 특성일 수 있어
  단정하기 어려움).

## 5. 참고 사항

- **SSG vs 동적 렌더링 재정리**: 코인 상세페이지(`coin-*.html`
  20개)는 완전 정적이라 SEO 안전. `index.html`/`coins.html`은
  이번 수정 전까지 "SSG로 만들어놓고 JS가 덮어쓰는" 자기 모순적
  구조였음 — 같은 문제가 재발하지 않도록, **앞으로 이 두 파일에
  상품 목록 관련 JS를 추가할 때는 반드시 `innerHTML` 통째 교체가
  아니라 기존 DOM 노드의 속성/텍스트만 갈아끼우는 방식으로 작성할 것.**
- **`data-premium` 속성의 역할**: SSG 빌드 시점에 시트의 premium
  값이 카드의 `data-premium` 속성으로 고정되어 들어감. 가격 계산은
  실시간 시세 × 이 속성값으로 이뤄지므로, premium을 시트에서
  바꿔도 Actions 재빌드 전까지는 옛 값으로 계산됨(의도된 동작).
- **`updateCardPricesFromSheet()`는 이미 3곳에 중복 정의돼 있었음**
  (`script.js`, `coins.html` 인라인, 각 `coin-*.html` 인라인) —
  이번 수정에서 로직 자체는 그대로 유지, 통합은 하지 않음(범위
  밖). 추후 공용 파일로 통합 고려 가능.
- **외부 SEO 검증의 한계**: 본인(Claude)은 구글 서치 콘솔에 직접
  접근할 수 없음. 웹 검색으로 사이트명/도메인을 검색해도 결과에
  안 잡히는 게 "색인 안 됨"의 확정적 증거는 아님(검색 API의 인덱스
  범위 한계일 수 있음) — 실제 색인 상태는 사용자가 서치 콘솔에서
  직접 확인해야 함.
