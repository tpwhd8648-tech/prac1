작성일: 2026-06-25

## 1. 목표

여러 세션째 보류 중이던 뱃지 shimmer 애니메이션 제거를 완료하고,
코인 추가/삭제 프로세스를 정리한다.

## 2. 현재 상태

### 2-1. 뱃지 shimmer 애니메이션 제거 (완료)

- **변경 파일**: `style.css`
- **변경 내용**: `@keyframes badge-shimmer`, `background-size`, `animation`,
  `background-image` 그라데이션 제거 → 정적 단색 `background`로 교체
  - IN STOCK: `#d4ae3a` (골드)
  - PRE SALE: `#d4bc70` (연골드)
- **충돌 검토**: `badge-instock`/`badge-presale` 스타일은 `style.css` 한 곳에만
  정의되어 있고, 다른 파일(`coins.html`, `index.html`, `coin-*.html`,
  `products.js`, `generate-coin-pages.js`, `coin-detail.html`)은 클래스명만
  사용 — 오버라이드 없음 확인.
- **push 완료**: 이번 세션 커밋.

### 2-2. 이번 세션에서 파악한 내용 (코드 변경 없음)

- **코인 추가/삭제 프로세스**: 시트 + `products.js` IMAGE_MAP +
  `coin-descriptions.js` 3곳이 keywords로 매칭되는 구조. 추가 시 3곳 모두
  작성 + 이미지 업로드 + push(Actions 자동 실행). 삭제는 `visible=none`으로
  그리드에서만 숨기거나, 완전 제거 시 시트 행 삭제 + 파일 직접 삭제.
- **`visible=none` 동작**: 그리드(index/coins)에서만 제거됨. 상세페이지
  (`coin-*.html`)는 URL 직접 접근 시 여전히 열림. sitemap에서는 제외되므로
  구글 신규 색인은 안 됨.
- **order 중복 시**: `a.order - b.order = 0`이 되어 시트 행 순서 그대로 유지.
- **coins.html 탭/정렬 필터**: 현재 PRE SALE 11개 중 10개가 premium=1(가격
  미설정)이고 IN STOCK이 5개뿐이라, 정렬/필터 버튼 추가가 실익 없다고 판단.
  가격이 다 채워지고 상품 수가 늘면 그때 추가 검토.
- **핸드오프 파일 수정**: `2026-06-25-1421` 파일의 "시트 변경 시 Actions
  재빌드 필요 인지" 항목을 To-Do에서 제거 (인지 완료, 작업 불필요).

## 5. 참고 사항

- **shimmer 애니메이션 복원 방법**: `style.css`의 `.badge-instock`,
  `.badge-presale` 블록에 아래를 다시 추가하면 됨:
  ```css
  @keyframes badge-shimmer {
    0%   { background-position: 200% center; }
    100% { background-position: -200% center; }
  }
  /* 공통 */
  background-size: 300% auto;
  animation: badge-shimmer linear infinite;
  /* instock */
  background-image: linear-gradient(90deg, #d4ae3a, #d4ae3a, #f5e080, #d4ae3a, #d4ae3a);
  animation-duration: 12s;
  /* presale */
  background-image: linear-gradient(90deg, #d4bc70, #d4bc70, #eada9e, #d4bc70, #d4bc70);
  animation-duration: 16s;
  ```
- **coins.html 탭 구조**: 현재 "금화" 탭 1개만 존재. 정렬/필터 버튼은
  상품 구성이 다양해지면 그때 추가 예정.
