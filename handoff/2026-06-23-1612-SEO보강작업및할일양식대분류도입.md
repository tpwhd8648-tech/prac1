## 2. 현재 상태

**완료 (push 반영 및 Actions 트리거, 직접 확인)**

- **`robots.txt`**: `Disallow: /mypage.html` 추가 (로그인 필요 페이지
  크롤링 차단). sitemap 위치는 기존에 이미 명시돼 있었음.
- **`BreadcrumbList` JSON-LD 추가**: 화면에 이미 있던 빵 부스러기
  UI(홈으로 / 금화 보기 / 코인명)와 동일한 경로를 구조화 데이터로 추가.
  `scripts/generate-coin-pages.js`와 `coin-maple-2026.html`(1단계 손작업
  템플릿) 양쪽에 적용.
- **메인 갤러리 이미지 alt 보강**: `alt="코인명"` → `alt="코인명 앞면"`으로
  변경. 썸네일도 `alt="앞면"`/`alt="뒷면"` → `alt="코인명 앞면"`/`alt="코인명
  뒷면"`으로 상품명을 포함시켜 일관성 확보.
  - **부수 발견 버그 수정**: `switchImage()`가 이미지 `src`만 바꾸고
    `alt`는 갱신하지 않아, 뒷면 클릭 후에도 alt 텍스트가 "앞면"으로
    남아있던 문제를 함께 수정 (alt도 src와 함께 갱신되도록 변경).
- **메인 product 이미지 preload/fetchpriority 추가**: `<head>`에
  `<link rel="preload" as="image" ... fetchpriority="high">` 추가,
  `<img>` 태그에도 `fetchpriority="high"` 속성 추가 (LCP 개선 목적).
- **검증 완료**: `node --check`로 JS 문법 확인, Python `json.loads`로
  Product/BreadcrumbList 양쪽 JSON-LD 문법 확인. 모두 통과.
- **커밋 `7976d82`**: 위 4가지 변경분 push 완료 (`coin-maple-2026.html`,
  `robots.txt`, `scripts/generate-coin-pages.js`).
- **GitHub Actions 수동 트리거 완료**: API로 `workflow_dispatch` 호출

## 5. 참고 사항

- **switchImage 버그 수정은 요청 범위 밖 추가 수정**: alt 텍스트 작업
  중 발견한 부수적 버그라 함께 고쳤음. 코드 검증(`node --check`)은
  문법만 확인하고 실제 시각적 동작은 보장하지 않으므로, 브라우저에서
  코인 상세페이지 갤러리 썸네일(특히 뒷면) 클릭해서 직접 확인 권장.
- **메이플 코인 페이지 특이사항**: `coin-maple-2026.html`은
  `generate-coin-pages.js`가 매일/수동 실행 시 자동으로 재생성하는
  파일이라(`coin-descriptions.js`에 메이플도 목록에 있음), 1단계
  손작업 템플릿이지만 실질적으로는 스크립트의 출력물 중 하나. 이번
  세션에서 두 파일(생성 스크립트 + 메이플 정적 파일) 모두 동기화해서
  수정함.
- **GitHub Actions 수동 트리거 방법(이번에 처음 API로 성공)**:
  ```
  POST /repos/{owner}/{repo}/actions/workflows/{workflow_id}/dispatches
  body: {"ref":"main"}
  ```
  워크플로우 ID는 `300577944`("Generate coin static pages"). PAT로
  curl 호출 시 204 응답이면 정상 트리거됨. 실행 상태는
  `/actions/runs/{run_id}`로 폴링해서 `status`/`conclusion` 확인.
  이번엔 워크플로우 파일 자체를 수정하지 않았으므로 일반 Contents
  Read/Write 권한 PAT로도 dispatch가 가능했음(workflow 파일 push와는
  다른 권한 요건).
- **로컬-원격 동기화 주의사항**: Actions가 자동 커밋을 만들면 원격이
  로컬보다 앞서가는 상태가 됨. 같은 세션에서 추가로 코드를 수정해 push할
  계획이 있다면 작업 전에 반드시 `git pull`로 동기화할 것 (이번 세션엔
  실제로 pull 후 작업 이어감, fast-forward로 충돌 없이 반영됨).
- **할일 양식 대분류 도입 배경**: SEO 관련 작업이 여러 세션에 걸쳐
  누적되며(검색노출, 메타태그, robots.txt, SSG, JSON-LD, 서치콘솔 등)
  색깔(긴급도) 기준만으로는 "SEO가 전체적으로 어디까지 왔는지"가 한눈에
  안 보이는 문제가 지적됨. 대분류를 1차 기준으로 바꾸고 색깔+담당자를
  각 줄에 보조 표시하는 방식(C안)으로 결정. 이번 인수인계 문서의 섹션
  4(To-Do)도 새 양식으로 작성함 — 앞으로 모든 핸드오프가 이 양식을
  따르게 됨.
- **대분류 정리 시 주의 (PROJECT_INSTRUCTIONS.md에 명시됨)**: 항목이
  1개뿐인 대분류도 소제목은 생략하지 않음. 여러 대분류에 걸치는 항목은
  더 핵심적인 대분류 하나에만 넣고 중복 등재하지 않음.
- **앞면/뒷면 기준**: 주제 그림(동물·건물 등) = 앞면(`.png`), 인물
  초상·국가 문장 = 뒷면(`-back.png`)
