# 카테고리 배너 이미지 인수인계 문서

작성일: 2026-06-19

## 1. 목표
index.html 메인 페이지의 카테고리 배너 3개(.cat-gold, .cat-silver, .cat-collectible)에 걸린 외부 unsplash 사진을 제거하고, 사이트 자체 그래픽(SVG 등)으로 대체한다. 외부 이미지 의존 제거로 로딩 속도 개선 및 브랜드 컬러 일관성 확보가 목적.

## 2. 현재 상태

### 확인 완료
- style.css 99~101행에서 .cat-gold/.cat-silver/.cat-collectible 모두 여전히 unsplash 외부 URL을 배경으로 사용 중임을 grep으로 확인 (코드 변경 없음, 기존 상태 그대로).
- index.html 124~135행 기준 각 배너의 실제 용도 확인: cat-gold→금화(coins.html), cat-silver→금 시세(gold-price.html), cat-collectible→문의(contact.html). (클래스명 cat-silver지만 실제 표시 문구는 "금 시세"로, 클래스명과 문구가 불일치하는 상태 — 이번 세션에서 수정하지 않음.)
- Claude는 포토리얼한 AI 이미지 생성 도구를 보유하고 있지 않음(이전 세션에서 "이미지 생성 가능"이라고 안내한 것은 부정확한 정보였음, 이번 세션에서 정정함). 실제 가능한 대안은 SVG/CSS 그래픽으로 직접 그리는 방식.
- 캔바(Canva) MCP 커넥터는 검색 결과 존재하나, 검색·기존 디자인 가져오기·자동 채우기·내보내기 위주 도구라 "제로베이스 신규 이미지 제작"에는 적합하지 않음. 이번 세션에서는 연동하지 않기로 함.

### 미완료
SVG 배너 시안 2차례 제작 (visualize 위젯으로 미리보기만 제공, 실제 코드/파일로는 저장하지 않음):
- v1: 금색/회색/초록 그라데이션 + 단순 동그라미·꺾은선·말풍선 아이콘 → 사용자 피드백: "전체적으로 밋밋함, 아이콘이 너무 단순함"
- v2: 메달리온(이중 테두리 + 8방향 엔그레이빙 선 + 잎 모티프), 차트 프레임 + 다이아몬드 포인트, 봉투 + 왁스실 디테일, 모서리 코너 장식, 비네트 + 사선 텍스처를 추가한 "고급스러운 미니멀" 방향 → 사용자 피드백: "다시 만들어야 할 것 같음, 보류"

두 시안 모두 채택되지 않음. **실제 레포 파일(style.css, index.html, images/)은 이번 세션에서 전혀 수정하지 않았음** — 미리보기만 보여주고 코드 반영 전 단계에서 중단.

## 3. 진행 단계

다음 세션에서 이어갈 경우:
1. (필수) 사용자에게 v1/v2의 구체적으로 어떤 점이 아쉬웠는지, 참고하고 싶은 레퍼런스 이미지/사이트가 있는지 먼저 물어본다. 두 차례 모두 막연한 "별로다" 피드백이었어서, 구체적 방향(색감 톤, 모티프 종류, 텍스트 배치 등) 합의 없이 세 번째 시안을 또 만들면 같은 결과가 반복될 가능성이 높다.
2. (필수) 합의된 방향으로 시안 제작 → visualize 위젯으로 미리보기 → 승인받은 후에만 style.css/이미지 파일에 실제 반영한다.
3. (선택, 반영 시 결정) SVG를 별도 파일(images/ 폴더)로 분리할지, style.css에 인라인 data URI로 넣을지 결정한다. 인라인은 외부 파일 요청이 아예 없어 더 빠르지만 CSS 파일 용량이 늘어난다.

## 5. 참고 사항

- **Claude는 포토리얼 이미지 생성 도구가 없음**: 이번 세션에서 확인된 중요한 사실. "이미지 생성해줘" 요청이 들어와도 실제로는 SVG/CSS 그래픽만 가능하다. 다음 세션에서 같은 요청이 들어오면 처음부터 이 점을 분명히 안내할 것.
- **시안 승인 프로세스가 비효율적이었음**: 막연한 "마음에 안 든다" 피드백만 받고 다음 시안을 만들면, 무엇이 바뀌었는지 사용자도 판단하기 어렵다. 다음에는 "어떤 점이, 왜" 안 맞는지 구체적 답변(색감/모티프/구도/톤 중 어디인지)을 먼저 받은 뒤 시안을 만드는 순서를 지킬 것.
- **클래스명-문구 불일치**: .cat-silver 클래스인데 실제 표시 문구는 "금 시세"임. 배너 이미지 작업과 별개로, 클래스명을 .cat-price 등으로 정리하면 나중에 혼동을 줄일 수 있다 (이번 세션에서는 손대지 않음).
- **레포 파일 변경 없음**: 이번 세션은 미리보기(visualize 위젯)만 진행했고 git에 push할 변경사항이 전혀 없다. 다음 세션 시작 시 별도 git pull/동기화 불필요.
- **이 문서는 push되지 않은 상태**: 이번 세션에 GitHub 토큰을 받지 못해 레포에 직접 push하지 못했다. 토큰을 주시면 이 문서를 handoff/ 폴더에 커밋·push할 수 있다.
