## 5. 참고 사항

- 모바일 헤더 검색 자리 이슈는 "코드에 빈 영역이 있다"는 인상과 달리,
  실제로는 이미 의도대로 숨겨져 있던 상태였음. 비슷하게 "뭔가 빠진 것
  같다"는 인수인계 To-Do는 실제 코드를 먼저 확인한 뒤 진짜 변경이
  필요한지 판단할 것 — 이번처럼 죽은 CSS 한 줄 정리로 끝나는 경우도
  있음.
- `fetchProductsFromSheet()`는 이제 3곳에서 호출됨: `script.js`
  (index.html, `visibleValues: ['all','main']`), `coins.html`
  (`visibleValues: ['all','coins']`), `coin-detail.html`
  (`includeAll: true`). 새 페이지에서 상품 데이터가 필요해지면 이 함수를
  재사용하고, 새로운 필터링 요구가 생기면 `options` 객체에 옵션을
  추가하는 방식으로 확장할 것(개별 파일에 파싱 로직을 새로 작성하지
  말 것).
- 구글시트 `상품` 탭과 `계산` 탭은 같은 `SHEET_ID`를 쓰지만 완전히
  다른 데이터(상품 목록 vs 실시간 시세)이고 코드도 분리되어 있음
  (`products.js`의 SHEET_ID는 `상품` 탭용, `nav.js`의 SHEET_ID는 `계산`
  탭용). 혼동해서 통합하려 하지 말 것 — 의도된 분리임.
- 레포 전체 기능 점검 시 점검한 항목: href/onclick 링크 무결성, id
  참조, alt 속성, 메타 태그(title/description), 함수 정의 여부. 모두
  정상이었고 이번에 발견된 이슈(중복 코드, 죽은 CSS)는 전부 처리됨.
- GitHub PAT 토큰: 이번 세션은 2회 모두 사용자가 매번 새로 채팅에
  입력해 push 진행, 재사용 없음. 세션 종료 후 두 토큰 모두 revoke
  권장.
