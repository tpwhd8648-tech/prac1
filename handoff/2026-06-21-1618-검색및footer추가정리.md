## 5. 참고 사항

- 검색 기능은 **데스크탑 전용**임(모바일 검색 UI 없음). `nav.js`의
  `goSearch(keyword)` 함수가 데스크탑 검색에서 쓰이고 있고, 모바일에 다시
  추가하게 되면 이 함수를 재사용하면 됨.
- `coins.html`의 `?q=` 필터 로직(`matchesKeyword`)은 한글 합성어 오매칭을
  막기 위해 키워드 앞뒤 글자가 한글이면 별개 단어로 보고 제외하는 방식임.
- footer.js의 "오시는 길" 링크는 더 이상 같은 페이지 내 앵커가 아니라
  `contact.html#contact-map`으로 다른 페이지의 지도로 이동함. 새로운
  페이지 내 앵커가 또 필요해지면 `id="footer-locations"`가 비어있는 채로
  남아있으니 재사용 가능.
- contact.html 지도 섹션(`#contact-map`)에는 sticky 헤더에 가려지지 않게
  `scroll-margin-top:130px`가 적용돼 있음. 헤더/네비 높이가 바뀌면 이 값도
  같이 조정해야 함.
- footer.js에서 삭제한 항목들은 전부 화면에는 안 보이지만 코드 내 HTML
  주석으로 "왜 지웠는지 + 실제 정보/페이지가 생기면 다시 추가하라"는 메모를
  남겨둠. footer.js를 열어보면 주석만 보고 어떤 항목이 "임시/영구 삭제"
  상태인지 바로 파악 가능함.
- GitHub PAT 토큰: 사용자가 매 push마다 채팅에 토큰을 평문으로 입력하다가,
  마지막엔 입력 없이 "푸쉬해줘"만 요청해서 대화에 이미 있던 토큰을 재사용해
  push함. push 자체는 매번 성공했지만, 세션 시작 시 매번 revoke 여부를
  확인해줄 것.
