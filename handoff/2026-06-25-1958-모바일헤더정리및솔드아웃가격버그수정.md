# 모바일 헤더 정리 및 솔드아웃 가격 버그 수정 인수인계 문서

작성일: 2026-06-25

## 1. 목표

모바일 헤더의 로그인 아이콘 중복을 제거하고, 솔드아웃(품절) 상품에 가격이
잘못 노출되던 버그들을 목록 카드·검색 결과·상세 페이지 전체에서 일관되게 수정한다.

## 2. 현재 상태

### 2-1. 완료 — 모바일 헤더 로그인 아이콘 중복 제거 (커밋 `e65899b`)

- 모바일(768px 이하)에서 햄버거 옆 사람 아이콘(`#auth-btn`, 로그인/마이페이지) 숨김
- 햄버거 메뉴 안에 이미 동일 기능의 "로그인/마이페이지" 링크(`#mobile-auth-link`)가
  있어서 중복이었음 → `style.css` 미디어쿼리에 `#auth-btn { display:none; }` 한 줄 추가
- 데스크탑(768px 초과)에는 영향 없음

### 2-2. 완료 — 솔드아웃 가격 노출 버그 3건 수정

**(1) coins.html 검색 결과 카드 — 뱃지/브랜드 깨짐 (커밋 `57ee5b0`)**
- 원인: 검색(`?q=`) 시에만 `products.js`의 `createProductCard()`로 그리드를 통째로
  다시 그리는데, 이 함수가 지난 `1928` 핸드오프(SOLD OUT 뱃지 개편) 때 같이 안
  고쳐져서 옛날 `soldout-overlay` div(CSS 삭제됨 → 안 보임)를 그대로 출력하고
  `data-brand` 속성도 없어서 브랜드 필터도 깨지는 상태였음
- 수정: `badge-soldout` span으로 통일 + `data-brand` 속성 추가

**(2) 목록 카드 가격이 시세 갱신 후 다시 보임 (커밋 `55c5b5d`)**
- 원인: SSG가 솔드아웃 카드에 "품절" 텍스트를 넣어주지만, `nav.js`가 30초마다
  부르는 `updateCardPricesFromSheet()`(`script.js`=index.html, `coins.html`
  인라인 두 곳 모두)가 솔드아웃 여부를 체크 안 하고 **모든** `.product-card`의
  가격을 무조건 덮어써서, 몇 초 뒤 "품절" → 실제 가격으로 바뀌어버렸음
- 수정: 두 함수 모두 `card.classList.contains('card-soldout')`이면 가격 갱신
  건너뛰도록 early return 추가

**(3) 상세 페이지에서 솔드아웃 상품 가격이 그대로 노출 (커밋 `9017a2d`)**
- 원인: 상세 페이지는 솔드아웃이어도 "판매가 ₩x,xxx,xxx"를 정상 계산해서
  그대로 보여주는 설계였음(목록 카드와 다른 동작) — 사용자가 직접 화면에서
  발견해 보고
- 수정: `scripts/generate-coin-pages.js` 템플릿에 `_isAvailable` 플래그 추가,
  솔드아웃이면 가격 영역도 "품절" 텍스트로 표시하도록 변경. 목록 카드와 동작 통일.
  - 템플릿(다음 빌드부터 자동 반영) + 그 시점 기준 솔드아웃 4개 정적 페이지
    (네스호·말띠·브리티시라이언·레이디저스티스) 직접 수정

### 2-3. 자동 빌드로 전체 20개 페이지에 (3) 반영 확인 완료 (커밋 `8885daa`, 자동)

- `generate-coin-pages.js`가 push 트리거 대상 파일이라 push 즉시 GitHub Actions가
  자동으로 돌아 20개 코인 페이지 전체를 재생성함
- 솔드아웃이 아니었던 `coin-maple-2026.html`에도 `_isAvailable` 로직이 정상
  포함된 것을 직접 확인함 (확인 완료 — 로컬에 `git pull` 후 grep으로 검증)

### 2-4. 논의만 하고 실행 안 한 사항

- 워크플로우(`generate-coin-pages.yml`)에는 `workflow_dispatch: {}`가 있어
  GitHub API로 토큰만 있으면 즉시 수동 트리거가 가능하다고 설명함
  (`POST /repos/.../actions/workflows/generate-coin-pages.yml/dispatches`)
- 사용자가 트리거를 원하는지 직접 실행 여부는 아직 안 물어봄 — 이번 세션
  마지막 메시지(인수인계+push 요청)로 대화가 끝나 실제 호출은 하지 않음

## 3. 진행 단계

1. (선택) 사용자가 시트에서 직접 솔드아웃 처리한 코인이 있다면, 자정 빌드까지
   기다리지 않고 `workflow_dispatch`로 즉시 트리거해줄지 다음 세션에서 확인.
2. (선택) 솔드아웃 상품의 가격을 상세페이지에서도 완전히 숨기는 게 맞는지,
   혹은 "이전 판매가" 같은 형태로 참고용으로 보여주는 게 나을지 운영 방향
   재확인 — 현재는 목록과 동일하게 완전히 숨기는 쪽으로 통일함.

## 5. 참고 사항

- **솔드아웃 관련 코드가 여러 곳에 분산돼 있어 한쪽만 고치는 실수가 반복됐음**:
  이번 세션에서 발견된 버그 3건 모두 "지난 세션에 한 파일만 고치고 동일 로직이
  있는 다른 파일(products.js, script.js, coins.html, 상세페이지 템플릿)은
  안 고쳐서" 생긴 것. 앞으로 솔드아웃/가격 관련 수정 시 아래 4곳을 항상 같이
  점검할 것:
  1. `scripts/generate-coin-pages.js` (SSG 템플릿 — 목록 카드 + 상세 페이지 둘 다 포함)
  2. `script.js` (index.html 런타임 가격 갱신)
  3. `coins.html` 인라인 스크립트 (런타임 가격 갱신 + 검색 시 재렌더링)
  4. `products.js` (coins.html 검색 시에만 쓰이는 카드 렌더 함수)
- **빌드 트리거 조건 재확인**: `.github/workflows/generate-coin-pages.yml`은
  `coin-descriptions.js`/`coin-data.js`/`products.js`/`scripts/generate-coin-pages.js`
  변경 push 시 즉시 실행, 그 외(시트 데이터만 변경 등)는 매일 00:00 KST 정기
  실행으로만 반영됨. `workflow_dispatch`로 수동 즉시 실행도 가능.
- **GitHub PAT는 이번 세션에서도 매번 사용자가 채팅에 직접 붙여 줌** — 토큰 자체를
  메모리/파일에 저장하지 않는 원칙 유지함.
