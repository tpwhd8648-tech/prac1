# 상품설명 탭 통일 및 시트 description 컬럼 제거 인수인계 문서

작성일: 2026-06-24

## 1. 목표

`coin-detail.html`(동적)과 정적 `coin-*.html` 20개 간 "상품설명" 탭 노출
내용이 서로 다르던 문제를 통일하고, 그 과정에서 발견된 미사용 시트
`description` 컬럼(전 행에 "추후 예정" placeholder만 들어있던 상태)과
빈 행 2개를 정리한다.

**상세 내용:**
- **발견 경위**: "금화 상품설명 탭 살펴봐줘" 요청으로 점검 중,
  `coin-detail.html`은 시트 `description`(짧은 설명) + `coin-descriptions.js`의
  `detail`(긴 설명) 둘 다 보여주는데, 정적 페이지는 `detail`만 보여줘서
  URL에 따라 화면이 달라지는 문제와, escape 처리 비대칭(정적은
  `escapeHtml()` 적용, 동적은 미적용)을 함께 발견함.
- **시트 description 실제 내용 확인**: 사용자가 시트 원본을 붙여넣어
  확인한 결과, 20개 행 전부 `"추후 예정"`이라는 미완료 placeholder
  텍스트가 들어있었음 (빈 칸이 아니었음). 이번 수정으로 이 문구가
  `coin-detail.html`에서도 더 이상 노출되지 않게 됨 — 의도치 않은
  placeholder 노출 문제가 부수적으로 해결됨.
- **빈 행 2개**: `2026 코뿔소 1oz`, `2026 세인트조지드래곤 1oz` — name과
  visible=none만 있고 나머지 컬럼 전부 공란. 사용자 확인 후 시트에서
  완전히 삭제.
- **컬럼 인덱스 변경 검증 방법**: description 삭제 전/후 시트 내용을
  Python으로 셀 단위 diff하여 "description 제외, 빈 행 2개 제외 시
  나머지 전부 100% 동일"함을 확인 후 진행. push 후 sitemap.xml의
  코인 URL 개수(20개)와 코뿔소/세인트조지드래곤 관련 URL 부재를
  grep으로 재확인.
- **행 순서 vs 열 순서**: 사용자 질문에 답하며 명확화함 — 코드는 시트를
  열(컬럼) 위치 인덱스로 읽으므로 **행(row) 순서는 자유롭게 바꿔도 무방**,
  **열(column) 순서/추가/삭제 시에만 코드(`products.js`의 인덱스) 동반
  수정 필요**. 이번 작업도 열 순서는 `name~order~image` 그대로 유지,
  description 열만 제거된 것이라 안전했음.

     인덱스 의존성을 grep으로 먼저 확인
  2. 열 삭제/추가 시 그 뒤에 있는 모든 컬럼의 인덱스가 한 칸씩
     밀리므로, 영향받는 모든 필드를 한 번에 같이 고칠 것
  3. 시트 수정과 코드 push 시점을 맞출 것 — 시트만 먼저 바꾸면 그
     사이 사이트가 깨진 상태로 노출됨 (이번엔 시트 수정 완료를 먼저
     확인받은 후 push해서 무중단 전환)
  4. **행(row) 순서는 코드에 영향 없음** — `order` 컬럼으로 별도 정렬하므로
     시트에서 행을 자유롭게 옮겨도 안전함. **열(column) 순서/구성
     변경만 코드 동반 수정이 필요**함을 사용자에게도 명확히 안내함.
- **`coin-descriptions.js`의 `detail` 필드가 이제 상품설명의 유일한
  소스**: 시트 `description` 컬럼이 삭제됐으므로, 앞으로 코인 상세
  설명 문구를 바꾸려면 시트가 아니라 `coin-descriptions.js`를 수정해야
  함 (코인별 `keywords`로 식별, 매칭 컬리전 주의는 기존 메모리 학습
