  fallback 전용 페이지인 `coin-detail.html`에서 canonical은

    핸드오프에서도 검토된 적이 없었던 별개의 미점검 항목이었음.
- **Twitter Card 메타태그 추가 (커밋 `59470f9`)**: `twitter:card`(
  `summary_large_image`)/`title`/`description`/`image` 4개 태그를
  기존 og: 값을 재사용해 추가.
  - 정적 페이지 6개: `index.html`, `coins.html`, `gold-price.html`,

  - `scripts/generate-coin-pages.js` 템플릿에도 추가
  - `coin-maple-2026.html` 동기화

  생성 스크립트 수정분이 코인 19개 페이지(메이플 제외)에 자동 반영됨.
  `coin-buffalo-2026.html`을 직접 가져와 4개 twitter: 태그 전부 정상
  반영 확인. 자동 커밋(`11f3c9a`) 발생 후 `git pull`로 로컬 동기화 완료.

**추가 점검 — 보류 결정 (코드 미반영)**

- **og:image:width/height/alt**: 추가를 검토했으나, (1) GitHub Actions
  빌드 환경이 `sparse-checkout`으로 `images` 폴더 전체를 제외하고
  있어 빌드 시점에 실제 이미지 크기를 읽을 방법이 없고, (2) 코인마다
  이미지 크기가 전부 다름(800×800 ~ 1192×1135 등 샘플 6개 확인 결과
  전부 다른 값)이 확인되어, 고정값을 넣으면 오히려 틀린 정보를 SNS
  크롤러에 전달하게 됨. 제대로 하려면 이미지 사이즈 통일 또는 빌드
  환경에 이미지 읽기 기능 추가가 필요해, 이번 세션 범위를 벗어나는
  별도 작업으로 To-Do에 남김.

**그 외 점검했으나 문제 없음으로 확인된 항목**:
- h1 태그 개수 (페이지마다 정확히 1개)
- lang="ko" / og:locale="ko_KR" 일관성
- og:type 구분 (상품 페이지 product / 나머지 website)
- WebSite JSON-LD (index.html에 정상 존재)
- meta description 길이 (전체 페이지 155자 이하)
- title 중복 (코인 20개 + 메인페이지 전부 고유)
- robots.txt ↔ sitemap.xml 일관성 (mypage.html 차단·제외 일치)

  핸드오프에 그렇게 적혀있나"를 직접 확인 요청해서 원문을 재검색한
  끝에 오류를 발견·정정함. 핸드오프 내용을 인용해 설명할 때는 실제
  원문을 grep으로 재확인하고 답하는 게 안전함 (이번처럼 비슷한 맥락의
  다른 항목과 혼동할 수 있음).
- **og:image 크기 불일치 데이터**: 샘플 확인한 6개 코인 이미지 크기 —
  메이플 1192×1135, 버팔로 1150×1131, 브리타니아 1168×1137, 캥거루
  1178×1145, 판다 800×800(유일하게 정사각형), 이글 1134×1129. 대부분

  이 데이터를 참고해 이미지 통일 여부를 판단할 것.
- **GitHub Actions sparse-checkout 제약**: `.github/workflows/
  generate-coin-pages.yml`의 체크아웃 단계가 `!images`로 이미지 폴더를
  제외하고 있어(용량 절약 목적), 빌드 스크립트(`generate-coin-pages.js`)

  검토해야 함.
- **GitHub Actions 수동 트리거 방법(재확인, 안정적으로 동작)**:
  ```
  POST /repos/{owner}/{repo}/actions/workflows/{workflow_id}/dispatches
  body: {"ref":"main"}
  ```
  워크플로우 ID `300577944`. PAT로 curl 호출 시 204 응답이면 정상
  트리거. 실행 상태는 `/actions/runs/{run_id}`로 폴링.
- **로컬-원격 동기화 주의사항**: Actions가 자동 커밋을 만들면 원격이
  로컬보다 앞서가는 상태가 됨. 이번에도 dispatch 직후 `git pull`로
  바로 동기화함 — 같은 세션에서 추가 작업이 필요하면 항상 먼저
  pull할 것.
- **앞면/뒷면 기준**: 주제 그림(동물·건물 등) = 앞면(`.png`), 인물
  초상·국가 문장 = 뒷면(`-back.png`)
