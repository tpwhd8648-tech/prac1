## 5. 참고 사항

- **시트는 git처럼 clone/push가 안 됨.** 사용자가 "구글시트도 git처럼
  자동화하고 싶다"고 문의했으나, 다음 이유로 "직접 셀 복붙" 방식으로
  최종 결정함:
  - 쓰기 자동화(Google Sheets API)는 Google Cloud Console에서 서비스
    계정 생성, JSON 키 발급, 시트에 그 계정 이메일 공유 등 사람이
    직접 브라우저에서 클릭해야 하는 설정이 필요해 Claude가 대신 할 수
    없음.
  - Claude의 bash 환경은 네트워크 허용 도메인이 제한적이라(github.com,
    pypi.org 등) `googleapis.com` 직접 호출도 불가능.
  - 빈도가 낮고(가끔 상품 추가/수정) 위험(키 노출, 자동 덮어쓰기 실수)
    대비 이득이 적어 직접 수정 방식이 더 안전하다고 판단.
  - **앞으로 운영 방식**: 사용자가 "OO 상품 추가/가격 변경해줘"라고
    요청하면, Claude가 시트 컬럼 순서(`name brand category premium
    available same_day visible order description image`)에 맞춰 탭
    구분 형태로 정확한 값을 제공하고, 사용자가 시트에 복사해 붙여넣는
    방식으로 진행하기로 함.

- **시트 헤더(1행) 텍스트는 자유롭게 바꿔도 코드에 영향 없음.** 코드
  (`products.js`)는 `row.c[0]`, `row.c[1]`처럼 **컬럼의 위치(순서)**로
  데이터를 읽지, 헤더 이름으로 읽지 않음. 즉 헤더 셀 문구를 한글로
  바꾸거나(예: `available` → `판매가능`) 다른 이름으로 바꿔도 컬럼
  순서만 유지하면 사이트는 그대로 동작함. 헤더명 변경 자체는 이번
  세션에서 실제로 진행하지는 않았음(필요 시 다음 세션에서 처리).

- **컬럼 의미 정리** (다음에 헤더명 바꿀 때 참고):
  - `available`: 판매 가능 여부. FALSE면 목록엔 보이지만 "품절"
    표시되고 구매 버튼 비활성화.
  - `same_day`: 당일출고 가능 여부. 이번 세션부터 "당일수령" 뱃지로
    실제 화면에 반영됨.
  - `visible`: 어느 페이지 목록에 노출할지(`all`/`coins`/`none` 등).
    재고와 무관하게 목록 노출 자체를 켜고 끄는 용도.

- `fetchProductsFromSheet()`(products.js)는 이제 3곳에서 호출됨:
  `script.js`(index.html, `visibleValues: ['all','main']`),
  `coins.html`(`visibleValues: ['all','coins']`), `coin-detail.html`
  (`includeAll: true`). 새 페이지가 필요해지면 이 함수를 재사용할 것.

- 뱃지류 CSS를 새로 추가할 때는 `index.html`/`coin-detail.html` 내부
  `<style>` 블록이 아니라 **`style.css`(공용 파일)에 추가**하는 것을
  우선 고려할 것 — 이번에 `index.html`에만 있던 죽은 뱃지 CSS가
  `coins.html`에는 없어서 페이지별로 따로 관리되고 있었던 사례가 있었음
  (이번에 정리함). 공용 스타일은 처음부터 `style.css`에 작성하면 향후
  비슷한 불일치를 막을 수 있음.

- GitHub PAT 토큰: 이번 세션은 1회 사용(당일수령 뱃지 커밋 push),
  매번 새로 채팅에 입력받아 진행, 재사용 없음. 세션 종료 후 revoke
  권장.
