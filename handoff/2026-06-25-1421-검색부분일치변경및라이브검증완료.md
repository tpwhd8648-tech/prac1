### 2-2. `script.js` stale 주석 수정 (완료, 이번 세션 커밋에 포함)

- **발견**: 이전 세션(`2026-06-24-1801`~`2026-06-25` 사이)에서
  `index.html`의 `<script src="products.js">`를 제거했는데, `script.js`에는
  "index.html에서 products.js를 script.js보다 먼저 로드해야 함"이라는
  이제는 사실과 맞지 않는 주석이 남아있었음.
- **확인**: `updateCardPricesFromSheet()`(script.js)는 실제로 `products.js`의
  어떤 함수도 호출하지 않아 의존성 자체가 없었음 — 제거가 원래부터 안전했고,
  주석만 stale 상태로 남아있던 것.
- **조치**: 주석을 "index.html은 더 이상 사용 안 함, coins.html/coin-*.html은
  여전히 사용 중" 으로 갱신. `node --check` 통과.

### 2-3. `coins.html` 검색 매칭 로직 변경 — 한글 합성어 경계 검사 완전 제거 (완료)

**배경**: 사용자가 라이브 검증 중 "메"만 입력하면 "메이플"이 검색 결과에
안 나오는 것을 발견. 원인은 기존 `matchesKeyword()`가 키워드 앞**과** 뒤
양쪽 글자가 한글이면 매칭을 제외하는 방식이었기 때문(이 로직은
`2026-06-21` 세션에 "라이언이글"에서 "이글"만 검색했을 때 오매칭되는 걸
막기 위해 도입됐던 것).

**시도 1 (중간 단계, 최종 미채택)**: "뒤" 글자 검사만 제거하고 "앞" 글자
검사는 유지하는 절충안을 구현·테스트함. "메"→메이플은 매칭되면서
"이글"→라이언이글 오매칭은 그대로 방지되는 것까지 git log로 원인 추적 +
20개 코인 전체 이름 대상 node 스크립트 테스트로 검증 완료.

**최종 결정**: 사용자가 절충안 대신 "코인이름 중 어떤 글자라도 매칭되면
검색 가능"하게 해달라고 명시적으로 요청 — 경계 검사를 **완전히 제거**한
단순 부분일치(`String.includes()`)로 변경. 이 경우 발생하는 트레이드오프
("이글" 검색 시 "이글" 코인뿐 아니라 "라이언이글"도 함께 노출, 흔한 글자
하나만 입력해도 관련 없는 코인이 여러 개 같이 걸릴 수 있음)를 사용자가
미리 안내받고 인지한 상태로 결정함.

**검증**: HTML 파싱(BeautifulSoup) 정상, 20개 코인 전체 이름 대상 node
스크립트로 "메"/"이글"/"라이언" 등 매칭 결과 확인, diff로 의도한 함수
내부만 변경됨 확인.

**Push 완료**: 커밋 해시는 본 문서 작성 후 push 시점에 확정(아래 참고).
