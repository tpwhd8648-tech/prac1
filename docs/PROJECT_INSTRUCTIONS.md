# 프로젝트 작업 지침 (Claude용)

이 문서는 `tpwhd8648-tech.github.io` 레포 작업 시 Claude가 따라야 할 워크플로우의
원본(단일 진실 소스)이다. 여러 계정에서 동일한 작업 지침을 쓰기 위해, 각 계정의
"프로젝트 지침" 칸에는 이 문서를 가리키는 짧은 안내만 두고, 실제 규칙은 여기서 관리한다.

이 문서를 수정하면 별도 계정 설정 변경 없이 모든 계정에 다음 대화부터 반영된다.

## 0. 공통: 모든 작업의 첫 단계 — 항상 먼저 실행

사용자가 코드 수정/확인 요청을 하면, 별도 요청 없이 항상 먼저 아래 명령으로 가볍게
받아온 뒤 작업을 시작한다:

```
git clone --filter=blob:none --no-checkout --depth 1 https://github.com/tpwhd8648-tech/tpwhd8648-tech.github.io.git tpwhd8648-tech.github.io && cd tpwhd8648-tech.github.io && git sparse-checkout init --cone && git sparse-checkout set --no-cone '/*' '!/images' && git checkout
```

(`images` 폴더는 용량만 크고 코드 작업엔 불필요하므로 제외)

## 1. 새 채팅 시작 시 — "할일뭐야?" 같은 질문을 받으면

할일 목록 정리 시 우선순위별로 색깔 이모지로 분류하고, 세부사항에 담당자를 표시한다:

- 🔴 긴급 / 즉시 처리
  - 🤖 버그 / 오류 수정
  - 🤖 코드 수정 필요
  - 👤 브라우저 확인 필요
- 🟡 일반 작업
  - 🤖 코드 수정 필요
  - 🤖 디자인 수정
  - 👤 데이터 / 시트 수정
- 🔵 보류
  - 👤 방향 결정 필요
  - 👤 외부 의존 (사용자 액션 대기)
- ⚪ 사실 확인
  - 👤 실제 사업 정보 (주소 / 연락처 등)
  - 🤖 코드 vs 실제 데이터 불일치 확인
- 📢 마케팅 / 노출
  - 👤 검색 노출 (네이버 / 구글)
  - 👤 SNS / 광고

(🤖 = Claude 처리 가능 / 👤 = 사용자 직접 액션 필요)



0번 실행 후, 레포 안 `/handoff/` 폴더(파일명 형식: `날짜-시간(HHMM)-주제.md`)에서
파일명 기준 가장 최근(날짜+시간 순) 파일을 찾아 읽고, 그 내용을 바탕으로 현재까지의
진행 상황, 남은 작업, 주의사항을 위 색깔 이모지 양식으로 요약해서 답변한다.
해당 폴더나 파일을 찾지 못하면 사용자에게 위치를 물어본다.

## 2. 코드 수정 요청 시

0번 실행 후, 수정된 부분만 주지 말고 해당 파일의 완성본 전체를 전달한다. 이때 한쪽만
고치고 다른 연관 파일을 놓치는 실수를 막기 위해, 별도 요청 없이 문법오류·함수/변수명
불일치·id-class 충돌·명백한 논리 오류를 grep/view로 빠르게 체크한 뒤 전달한다.

## 3. GitHub 토큰을 받았을 때 — push 워크플로우

사용자가 채팅에 `github_pat_...` 형태의 토큰을 붙여넣으면:

1. 토큰으로 해당 레포에 대한 읽기/쓰기 권한을 조용히 확인한다 (화면에 토큰 값을
   다시 출력하지 않음).
2. 코드 수정이 끝나면 변경 사항을 요약해서 보여주고, 반드시 사용자에게
   "푸쉬해줘" 같은 명시적 확인을 받은 후에만 push한다. 확인 없이 자동으로
   push하지 않는다.
3. push 후에는 `git log` 또는 `git ls-remote`로 반영이 잘 됐는지 확인하고,
   커밋 해시와 결과를 사용자에게 보고한다.
4. 토큰은 절대 메모리(memory_user_edits)나 파일, 로그에 저장하지 않는다.
   해당 세션의 작업에만 사용하고 응답에 다시 노출하지 않는다.

## 4. 작업 마무리 시 — "인수인계 만들어줘" 요청을 받으면

handoff-doc 스킬(`/mnt/skills/user/handoff-doc/SKILL.md`)의 문서 구조(목표/현재상태/
진행단계/To-Do/참고사항 5개 섹션)와 작성 유의사항은 그대로 따른다.

**단, To-Do 섹션(4번)은 스킬의 체크박스(`- [ ]`) 양식 대신 이 문서 섹션 1의
색깔 이모지 + 담당자(🤖/👤) 양식을 우선한다.**

다만 **파일명 규칙은 이 문서가 우선한다**: 스킬 파일에 적힌 형식과 다르더라도,
항상 `YYYY-MM-DD-HHMM-주제.md` (날짜+시간(HHMM, 24시간제)+주제) 형식을 사용해
`/handoff/` 폴더에 저장한다. 같은 날 여러 작업을 처리할 수 있어 시간까지 포함해야
파일명이 겹치지 않고 작성 순서가 명확해지기 때문이다.

**타임스탬프는 반드시 한국시간(KST, UTC+9) 기준으로 통일한다.** 작업 컨테이너는
UTC로 동작하므로, bash의 `date` 명령을 그냥 쓰면 UTC 시각이 나와 날짜가 하루
어긋날 수 있다. 파일명에 쓸 날짜+시간을 구할 때는 아래처럼 명시적으로 KST로
환산해서 얻는다:

```
TZ='Asia/Seoul' date '+%Y-%m-%d-%H%M'
```

날짜(YYYY-MM-DD)와 시간(HHMM)을 서로 다른 방법(예: 대화 맥락상 "오늘 날짜"는
KST로 알고 있으면서 시간만 별도로 UTC `date` 명령으로 구하는 식)으로 섞어서
조합하지 않는다 — 한쪽은 KST, 한쪽은 UTC가 되어 날짜·시간이 어긋나는 사고가
실제로 있었다 (2026-06-21 관련 핸드오프 문서 3건의 날짜/시간 오기, 이후 수정됨).

이번 세션에서 진행한 작업 내용, 변경된 파일 목록, 남은 작업, 다음 세션에서 참고할
사항을 포함한다.

**To-Do 섹션은 섹션 1의 색깔 이모지 + 담당자 양식으로 작성한다.** 체크박스(`- [ ]`)
대신 해당 색깔 이모지와 🤖/👤 담당자 아이콘을 붙여 항목을 작성한다. 예시:

```
### 🔴 긴급 / 즉시 처리
- 👤 브라우저에서 뱃지 노출 확인

### 🟡 일반 작업
- 🤖 footer.js 더미 데이터 정리
```
